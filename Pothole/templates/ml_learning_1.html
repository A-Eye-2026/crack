🐟 세션 1: KNN으로 생선 분류하기

1-1. K-최근접 이웃 (K-Nearest Neighbors)

가장 단순하면서도 강력한 분류 알고리즘이에요. 새로운 데이터가 들어오면 "너 누구랑 가장 비슷해?"라고 물어보는 방식이죠.

  • 📏 거리 계산: 좌표 평면에서 가장 가까운 이웃 5개를 찾아요.
  • 🗳️ 다수결의 원칙: 이웃 5개 중 도미가 많으면 도미, 빙어가 많으면 빙어로 결정!
  • 🧠 특징(Feature): 생선의 '길이'와 '무게'가 분류의 기준이 됩니다.
💡 심화 학습: 단순히 가깝다고 다 같은 이웃일까요?
데이터의 단위(cm vs g)가 너무 다르면 그래프상 거리가 왜곡될 수 있어요. 이를 해결하는 방법은 3강에서 배울 '표준화'입니다!

🎯 KNN, 단어의 의미를 파헤쳐보자! (Mastery Note)

K K (수/개수)

몇 명의 이웃을 볼 것인가? K=5면 가장 가까운 5명을, K=1이면 단 한 명의 말만 듣겠다는 뜻입니다.

N Nearest (가장 가까운)

거리가 가장 짧은 위치를 찾습니다. '유클리드 거리'라는 수학적 공식을 사용하여 직선 거리를 계산해요.

N Neighbors (이웃들)

주변 이웃들의 정답(레이블)을 확인합니다. 내 주변에 도미가 많으면 나도 도미가 되는 식이죠.

🚩 한 줄 요약: "새로운 데이터가 들어오면, 가장 가까운(Nearest) K명의 이웃(Neighbors)에게 물어보고 다수결로 결정한다!"
✨ 좋은 학습 vs 나쁜 학습 비교
❌ 나쁜 예 (나홀로 데이터)

데이터가 1~2개뿐이면, 새로운 생선이 왔을 때 엉뚱한 이웃을 참조하게 되어 오답률이 급격히 올라가요.

✅ 좋은 예 (풍부한 데이터)

다양한 크기의 도미/빙어 데이터가 골고루 있으면, 경계선 근처의 데이터도 아주 정확하게 맞출 수 있습니다.

💻 코드 ① — 데이터 준비
import numpy as np

# 도미와 빙어 데이터를 합쳐서 하나의 리스트로 만듭니다.
fish_length = bream_length + smelt_length
fish_weight = bream_weight + smelt_weight

# [길이, 무게]가 한 쌍이 되도록 2차원 배열로 변환합니다.
fish_data = np.column_stack((fish_length, fish_weight))
fish_target = np.array([1]*35 + [0]*14)
핵심: column_stack()은 두 배열을 나란히 세운 뒤 행 단위로 묶어줍니다.
💻 코드 ② — 시각화 (산점도)
import matplotlib.pyplot as plt

plt.scatter(bream_length, bream_weight, label='bream')
plt.scatter(smelt_length, smelt_weight, label='smelt')
plt.xlabel('length')
plt.ylabel('weight')
plt.legend()
plt.show()
핵심: 데이터를 그래프에 점으로 찍어보면 도미와 빙어의 특징이 뚜렷하게 구분됩니다.
💻 코드 ③ — 모델 생성
from sklearn.neighbors import KNeighborsClassifier

# 모델 객체를 생성합니다. 이웃 개수는 기본값인 5로 설정합니다.
kn = KNeighborsClassifier(n_neighbors=5)
핵심: 사이킷런 모델은 클래스를 객체로 먼저 만들어야 사용 가능합니다.
💻 코드 ④ — 모델 학습 (fit)
# fit() 메서드는 준비한 데이터로 모델을 훈련시킵니다.
kn.fit(fish_data, fish_target)
핵심: 훈련이란, 데이터 간의 관계나 특성을 모델이 기억하게 하는 과정입니다.
💻 코드 ⑤ — 성능 평가 (score)
# 모델이 얼마나 잘 맞히는지 확인해봅니다 (0.0 ~ 1.0).
score = kn.score(fish_data, fish_target)
print(f"정확도: {score:.4f}")
핵심: score()는 내부적으로 predict를 수행한 뒤 실제 정답과 비교합니다.
💻 코드 ⑥ — 새로운 데이터 예측
# 길이 30cm, 무게 600g인 생선은 무엇으로 판정할까요?
prediction = kn.predict([[30, 600]])
print("예측 결과:", prediction) # [1] -> 도미!
핵심: predict()를 쓸 때는 입력 데이터를 반드시 리스트의 리스트(2차원)로 줘야 합니다.
📗 용어 사전
KNN (K-Nearest Neighbors): "가장 가까운 K명에게 물어보기" 알고리즘
분류 (Classification): 데이터를 정해진 그룹(예: 도미/빙어)으로 나누는 것

❓ 궁금할 수 있는 질문들

Q: K값을 너무 작게(예: 1) 설정하면 어떻게 되나요?

A: 데이터의 아주 작은 노이즈에도 민감하게 반응하여 모델이 너무 예민해집니다(과대적합 위험). K=1이면 바로 옆집 말만 듣는 고집불통이 되죠.

Q: K값을 너무 크게(예: 전체 데이터 개수) 설정하면요?

A: 모든 예측을 "주류 정답"으로만 하게 됩니다. 도미가 35개, 빙어가 14개라면 K=49일 때 무조건 '도미'로만 대답하게 되는 거죠.

Q: 왜 모델 훈련(fit) 이후에 score가 1.0이 나왔는데 함정인가요?

A: 시험 범위를 다 가르쳐주고 똑같은 문제로 시험을 본 것과 같습니다. 실전(안 배운 데이터)에서도 잘할지는 알 수 없으니까요!

Q: KNN은 실무에서 많이 쓰이나요?

A: 데이터가 아주 많아지면 모든 거리 계산이 너무 오래 걸려서 잘 쓰지 않아요. 하지만 작고 직관적인 데이터에서는 아주 훌륭한 출발점이 됩니다.

Q: 유클리드 거리 말고 다른 방식도 있나요?

A: 네, 맨해튼 거리(격자 형태 거리)나 코사인 유사도(방향성 중심) 등 다양합니다. 하지만 분류 문제에서는 직선 거리인 유클리드를 가장 많이 씁니다.

Q: 생선의 특징을 더 많이(예: 높이, 두께) 추가하면 더 똑똑해지나요?

A: 일반적으로는 그렇습니다. 하지만 관련 없는 정보가 섞이면 오히려 헷갈려할 수도 있어요. 이를 '차원의 저주'라고 부르기도 합니다.
