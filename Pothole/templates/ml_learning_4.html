📊 세션 4: K-최근접 이웃 회귀

4-1. 분류 vs 회귀

지금까지는 "이게 뭐야?(분류)"를 물어봤다면, 이번에는 "이게 얼마야?(회귀)"를 물어봅니다.

  • 🎯 회귀(Regression): 임의의 어떠한 숫자를 예측하는 작업 (예: 몸무게, 집값, 주가)
  • ⭐ 결정계수(R²): 예측이 실제와 얼마나 가까운지 나타내는 지표 (1에 가까울수록 완벽!)

🎯 결정계수 ($R^2$)란?

분류에서는 '정확도'를 따졌지만, 회귀에서는 숫자가 딱 맞기 힘들기 때문에 결정계수를 사용합니다.

$$ R^2 = 1 - \frac{\sum(\text{타깃} - \text{예측})^2}{\sum(\text{타깃} - \text{평균})^2} $$

점수가 0.99라면 실제 값의 차이(오차)가 평균과의 차이보다 매우 작다는 뜻으로, 아주 잘 예측하고 있다는 의미입니다.

✨ 회귀의 정확도 판단
❌ 무지성 평균 예측 (Bad)

모든 농어의 무게를 단순히 '전체 평균값'으로 대답한다면 분모와 분자가 같아져 $R^2$은 0이 됩니다. 당연히 실무에선 쓸모 없는 모델입니다.

✅ 이웃의 평균 (Good)

주변 K개의 가까운 이웃들(크기가 비슷한 농어)의 무게 평균을 내면 실제 몸무게에 근접하여 양수의 높은 점수가 나옵니다.

주의: K값에 따른 과대적합과 과소적합

💡 학계 노트: K(이웃의 수)가 미치는 영향

  • K=1인 경우: 가장 가까운 딱 하나의 이웃의 값만 맹신하게 되어 곡선이 지진계처럼 널뛰게 됩니다. 이는 훈련 세트의 작은 잡음까지 모두 수용해 버리는 과대적합(Overfitting) 상태입니다. 모델이 훈련 데이터에 너무 민감하게 반응하여 일반적인 패턴을 놓치게 됩니다.
  • K=42인 경우: 전체 농어(훈련 세트 42마리)를 다 이웃으로 삼아서 예측을 뭉개버립니다. 즉, 거대한 100cm 농어가 와도, 작은 10cm 빙어가 와도 무조건 전체 평균 무게를 답합니다. 선이 완전히 평평해지며 과소적합(Underfitting) 상태가 됩니다.
💻 코드 ① — 농어 데이터 준비
import numpy as np

# 농어의 길이와 무게 데이터를 준비합니다.
perch_length = np.array([8.4, 13.7, 15.0, ...])
perch_weight = np.array([5.9, 32.0, 40.0, ...])
핵심: 회귀 분석을 위해 입력(길이)과 타깃(무게)을 넘파이 배열로 만듭니다.
💻 코드 ② — 데이터 모양 바꾸기 (reshape)
# 사이킷런에서 사용할 수 있도록 1차원 배열을 2차원(n, 1)으로 바꿉니다.
train_input = train_input.reshape(-1, 1)
test_input = test_input.reshape(-1, 1)
핵심: reshape(-1, 1)은 행의 개수는 알아서 맞추고 고정된 1개 열로 만들라는 뜻입니다.
reshape(-1, 1)의 마법
왜 굳이 모양을 바꾸나요?

사이킷런 같은 AI 모델은 데이터를 받을 때 "샘플 하나당 특징 한 줄"인 2차원 표 형태를 기대합니다. 하지만 우리가 처음 만든 데이터는 단순히 숫자들을 나열한 1차원 리스트 형태죠.

  • 1: "무조건 1개의 열(특징)을 가진 표로 만들어줘!"라는 뜻입니다.
  • -1: "데이터가 총 몇 개인지는 네가 알아서 세어서 행을 채워줘!"라는 뜻입니다.
바보 예방 팁

데이터가 42개라면 자동으로 (42, 1)이 됩니다. AI에게 "입체 사진 더미"가 아닌 "정돈된 표"를 주는 것이 핵심입니다!

💻 코드 ③ — 모델 생성 및 훈련
from sklearn.neighbors import KNeighborsRegressor

# 분류가 아닌 회귀(Regressor) 기능을 사용합니다. (K의 기본값인 5가 사용됨)
knr = KNeighborsRegressor()
knr.fit(train_input, train_target)
핵심: 분류 모델(KNeighborsClassifier)과 내부 알고리즘 뼈대가 같아 사용법이 동일하지만, 이웃들의 '클래스' 대신 '수치값'의 평균을 구합니다.
💻 코드 ④ — 결정계수 (R²) 확인
print(knr.score(test_input, test_target))
# 결과: 0.9928...
핵심: 숫자가 1에 가까울수록 모델이 데이터를 아주 정확하게 설명하고 있다는 뜻입니다.
💻 코드 ⑤ — 절대 평균 오차 (MAE, Mean Absolute Error)
from sklearn.metrics import mean_absolute_error

# 모델이 알아낸 예측값과 정답 간의 평균적인 차이를 구합니다.
test_prediction = knr.predict(test_input)
mae = mean_absolute_error(test_target, test_prediction)
print(mae)
핵심: MAE는 타깃과 예측값의 오차들에서 부호(+, -)를 떼고 절댓값을 취해 평균낸 것입니다. 모델이 평균적으로 '19g' 정도를 빗나갔다는 구체적인 직감을 제공합니다 (R²와 달리 측정 단위가 존재함).
💻 코드 ⑥ — 과대적합 vs 과소적합 확인 (맛보기)
print(knr.score(train_input, train_target))
print(knr.score(test_input, test_target))
핵심: 훈련 세트보다 테스트 세트 점수가 높거나 두 점수가 모두 낮으면 '과소적합'을 의심해야 합니다.

❓ 궁금할 수 있는 질문들

Q: 결정계수(R²)가 1이면 무조건 최고인가요?

A: 이론적으로는 좋지만, 훈련 데이터에만 너무 맞춘 나머지 새로운 데이터에는 엉뚱한 답을 낼 수도 있어요(과대적합). 항상 테스트 세트 점수와 비교해야 합니다.

Q: 분류와 회귀를 어떻게 구분하나요?

A: "A인가 B인가?"를 맞추면 분류, "얼마(숫자)인가?"를 맞추면 회귀입니다.

Q: MAE와 MSE는 어떤 차이가 있나요?

A: MAE는 단순한 거리 차이이고, MSE는 오차를 제곱하여 합친 것입니다. MSE는 큰 오차일수록 더 큰 벌칙을 주는 효과가 있습니다.

Q: K값이 작을수록 회귀 곡선이 어떻게 변하나요?

A: K가 작아질수록 훈련 데이터 하나하나에 민감하게 반응하여 곡선이 매우 구불구불해집니다. 반대로 K가 커지면 아주 부드러운 선이 됩니다.

Q: 왜 회귀에서는 정확도(Accuracy)라고 안 하고 결정계수라고 하나요?

A: 연속적인 숫자를 맞추는 것이라 "정확히 맞았다/틀렸다"를 정의하기 어렵기 때문입니다. 대신 실제 값의 변동을 모델이 얼마나 잘 '설명'하는지 비율로 나타내는 것입니다.
