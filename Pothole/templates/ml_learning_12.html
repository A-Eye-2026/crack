스스로 군집의 중심을 찾는 k-평균

k-평균(k-means)은 비지도 학습의 대표적인 알고리즘으로, 데이터의 평균값을 자동으로 찾아 줍니다. 이 평균값은 클러스터의 중심에 위치하기 때문에 센트로이드(centroid)라고 부릅니다.

왕초보를 위한 3단계 요약

1. 무엇을(What)?

이름표 없는 사진 수백 장을 컴퓨터에게 줍니다.

2. 언제(When)?

데이터가 너무 많아 일일이 분류하기 힘들 때 사용합니다.

3. 왜(Why)?

비슷한 것끼리 '알아서' 묶어주는 마법이 필요하니까요!

지도 학습 (지금까지 배운 것)

"이건 사과야, 이건 바나나야"라고 정답을 알려주고 공부시킵니다.
정해진 이름표(라벨)를 맞히는 것이 목표예요. (마치 답지가 있는 문제집 풀기!)

비지도 학습 (오늘 배우는 것)

정답(이름표)이 아예 없습니다.
컴퓨터가 "이것들은 비슷하게 생겼네?" 하고 스스로 분류합니다. (마치 답지 없이 보물찾기 하는 것과 같죠!)

어렵게 생각하지 마세요! 수천 개의 장난감이 섞여 있을 때, "색깔이나 모양이 비슷한 애들끼리 3개 주머니에 나눠 담아줘!"라고 명령하는 것과 같습니다.

작동 원리 (Step-by-Step):
  1. 무작위로 k개의 클러스터 중심(센트로이드)을 정합니다.
  2. 각 데이터에서 가장 가까운 센트로이드를 찾아 해당 클러스터의 샘플로 지정합니다.
  3. 클러스터에 속한 샘플들의 실제 평균값으로 센트로이드를 이동시킵니다.
  4. 중심점에 변화가 없을 때까지 2~3번 과정을 반복하여 최적의 군집을 완성합니다.
엘보우(Elbow) 방법?

최적의 k(군집 개수)를 찾는 방법입니다. 군집 개수를 늘려가며 이너셔(Inertia, 중심과 샘플 거리의 제곱 합)를 측정했을 때, 줄어드는 속도가 꺾이는 지점(사람의 팔꿈치 모양)이 가장 적절한 k값입니다.

알고리즘이 찾은 군집의 중심 (Centroids)

k-평균 알고리즘이 300장의 과일 사진을 분석하여 스스로 찾아낸 3개의 중심 이미지입니다.
사과, 파인애플, 바나나의 전형적인 특징을 잘 추출해낸 것을 볼 수 있습니다.

컴퓨터가 300장의 사진을 분석해 찾아낸 '사과, 바나나, 파인애플'의 평균 모습(Centroids)입니다.

잠깐! 클러스터 개수(k)를 잘못 설정하면 어떤 그림이 그려질까요?

🔍 너무 작게 설정했을 때 (k=2)

사과와 파인애플이 비슷하게 생겼다고 판단하여 한 팀으로 묶어버립니다.

이미지 결과: 사과와 파인애플이 겹쳐진 '희미한 혼합 이미지'가 그려집니다. 바나나만 겨우 독립된 그룹을 유지하게 되죠.

🔍 너무 크게 설정했을 때 (k=5, 10)

같은 사과 그룹 안에서도 "조금 더 빨간 것"과 "덜 빨간 것"을 다른 팀으로 쪼갭니다.

이미지 결과: 똑같은 사과 모양의 센트로이드가 2~3개씩 중복해서 나타납니다. 데이터가 너무 잘게 쪼개져 실질적인 의미가 퇴색되죠.
reshape(-1, 100*100)의 마법
왜 평평하게(Flatten) 만드나요?

처음 fruits는 (300, 100, 100) 모양의 입체적인 데이터입니다. 하지만 K-Means 알고리즘은 "한 행이 사진 한 장"인 2차원 표 데이터를 원합니다.

  • 100*100 (10,000): 가로·세로 픽셀을 한 줄로 길게 늘어뜨린다는 뜻입니다.
  • -1: "데이터 개수에 맞춰서 행 개수(사진 수)는 알아서 계산해!"라는 똑똑한 옵션입니다.
결과물 예시

데이터는 자동으로 (300, 10000) 모양이 됩니다. AI에게 "입체 더미"가 아닌 "정돈된 표"를 주는 것이 분석의 핵심입니다!

코드 한 줄 분석: K-평균 실행하기
# 1. 2차원(100x100) 이미지를 1차원(1만 개) 목록으로 펼치기
fruits_2d = fruits.reshape(-1, 100*100)
# 2. "3개로 묶어줘!" 모델 만들기 (랜덤 시드 42 고정)
km = KMeans(n_clusters=3, random_state=42)
# 3. 데이터 학습 (비지도 학습이므로 타겟 없이 스스로 패턴 찾기)
km.fit(fruits_2d)
# 4. 군집 결과 확인 (0, 1, 2 등의 숫자로 라벨링됨)
print(km.labels_)
  • reshape: 컴퓨터는 2차원 그림보다 한 줄로 길게 늘어선 숫자 목록을 더 잘 처리합니다. 1만 개의 픽셀을 한 줄로 세우는 과정입니다.
  • n_clusters=3: 우리가 이미 과일이 3종류라는 것을 알고 있기에 3으로 설정했습니다.
  • fit: 이 명령을 내리는 순간, 컴퓨터가 300장의 사진을 수없이 비교하며 비슷한 것끼리 묶기 위한 최적의 중심점을 찾기 시작합니다.
분석 결과: 컴퓨터가 찾은 '가장 과일다운' 이미지

1. 어떻게 만들어졌나요?

이 이미지들은 실제 사진이 아니라, 각 군집에 속한 100장의 사진을 픽셀별로 모두 평균 낸 결과입니다. 사과는 둥글게, 바나나는 길쭉하게 평균이 잡히면서 각 과일의 '본질적인 형태'만 남게 된 것이죠.

2. 코드로 들여다보기: 센트로이드(Centroid)의 비밀

km.cluster_centers_

이 속성에는 300장의 사진을 분석해 얻어낸 3개의 핵심 평균 데이터가 들어있습니다. 우리가 이 값을 시각화하면 위 사진처럼 과일의 뽀얀 평균 모습이 나타나는 것입니다. 머신러닝이 스스로 학습한 '지식'의 실체라고 볼 수 있습니다.

3. 센트로이드를 눈으로 보는 시각화 코드

# 3개의 센트로이드를 나란히 보여주는 그래프 만들기
fig, axes = plt.subplots(1, 3, figsize=(15, 5))
# 각 센트로이드를 하나씩 꺼내서 그리기
for i in range(3):
  axes[i].imshow(
    km.cluster_centers_[i].reshape(100, 100),
    cmap='gray_r')
  • plt.subplots(1, 3): 1행 3열의 빈 캔버스를 만듭니다. 3개의 센트로이드를 나란히 보여주기 위한 '액자 틀'이에요.
  • cluster_centers_[i]: i번째 군집의 중심(센트로이드) 데이터를 꺼냅니다. 이것은 1만 개의 숫자로 이루어진 1차원 배열이에요.
  • .reshape(100, 100): 아까 1줄로 펼쳤던 데이터를 다시 100×100 정사각형 이미지로 되돌립니다. (펼치기 ↔ 되감기)
  • cmap='gray_r': 색상표(colormap) 설정입니다. gray_r은 "반전된 회색"으로, 숫자가 클수록 밝게(하얗게) 표시됩니다. 과일의 밝은 부분이 흰색으로 나타나 직관적이에요.
엘보우 방법: 최적의 K 찾기

클러스터 개수(k)가 늘어날수록 이너셔 값은 자연스럽게 줄어듭니다.
이 그래프에서는 k=3 지점에서 기울기가 급격히 변하는 '팔꿈치' 구간이 정답임을 상징합니다.

그래프 정밀 분석: 왜 3이 정답일까요?

1. 축의 의미

  • x축 (Number of Clusters): 나눌 군집의 개수(k)입니다.
  • y축 (Inertia): 각 데이터가 소속된 중심과 얼마나 멀리 떨어져 있는지 나타내는 수치입니다. (낮을수록 좋습니다.)

2. 분석 결과

군집 개수가 늘어날수록 이너셔는 줄어들지만, 3 이후로는 줄어드는 속도가 눈에 띄게 완만해집니다. 더 쪼개봐야 큰 이득이 없다는 뜻이죠. 그래서 꺾이는 지점인 '3'을 최적의 개수로 판단합니다.

💡 코드로 보는 이너셔(Inertia):

km.inertia_

이너셔는 "각 데이터가 자기 팀의 중심(센트로이드)과 얼마나 가까이 뭉쳐있는가"를 나타내는 점수입니다. 점수가 작을수록 아주 빽빽하게 잘 모였다는 뜻이죠! 하지만 무작정 팀을 많이 늘리면(k가 커지면) 당연히 점수는 낮아지기 때문에, 효율이 급격히 떨어지는 '엘보우' 지점을 찾는 것이 중요합니다.

3. 엘보우 그래프를 그리는 전체 코드

# 이너셔를 담을 빈 리스트
inertia = []
# k를 2부터 6까지 바꿔가며 실험
for k in range(2, 7):
  km = KMeans(n_clusters=k)
  km.fit(fruits_2d)
  inertia.append(km.inertia_)
# 결과를 꺾은선 그래프로 시각화
plt.plot(range(2, 7), inertia)
  • inertia = []: 각 k에서 나온 이너셔 점수를 저장할 빈 그릇을 미리 준비합니다.
  • for k in range(2, 7): k=2, 3, 4, 5, 6으로 5번 반복 실험합니다. "2팀으로 나누면?", "3팀으로 나누면?" ... 계속 시도해 보는 것이죠.
  • km.fit(fruits_2d): 매 k마다 새로운 K-평균 모델을 만들고 학습시킵니다.
  • inertia.append(km.inertia_): 그 결과로 나온 이너셔(뭉침 점수)를 리스트에 하나씩 추가합니다.
  • plt.plot(...): x축에 k값, y축에 이너셔 점수를 놓고 꺾은선 그래프를 그립니다. 이 선이 꺾이는 지점이 바로 '엘보우(팔꿈치)'입니다!
보너스: 시각화 도우미 `draw_fruits` 함수 이해하기

과일 사진 수백 장을 격자 형태로 예쁘게 그려주는 마법 같은 함수입니다. 한 줄씩 뜯어볼까요?

def draw_fruits(arr, ratio=1):
    n = len(arr) # ① 사진 개수 확인
    rows = int(np.ceil(n/10)) # ② 몇 줄이 필요할까? (반올림)
    cols = n if rows < 2 else 10 # ③ 10개보다 작으면 그만큼만, 아니면 10칸
    fig, axs = plt.subplots(rows, cols, figsize=(cols*ratio, rows*ratio), squeeze=False)
    for i in range(rows):
        for j in range(cols):
            if i*10 + j < n: # ④ 사진이 남아있을 때만 그리기
                axs[i, j].imshow(arr[i*10 + j], cmap='gray_r')
            axs[i, j].axis('off') # ⑤ 눈금은 필요 없으니 끄기!
① np.ceil(n/10)의 수학적 의미

사진 35장을 그리려면 3.5줄이 아니라 4줄이 필요하겠죠? ceil은 소수점이 있으면 무조건 올림하여 필요한 방(Row)을 넉넉히 예약합니다.

② i*10 + j 인덱싱 기법

2차원 아파트(Row, Col)를 1차원 목록(Index)으로 변환하는 마법 공식입니다. i행 j열에 위치한 사진이 원본 목록의 몇 번째인지 정확히 찾아내어 imshow에 전달합니다.

🧪 클러스터링 결과 탐색기

k-평균 알고리즘이 300장의 과일 사진을 3개의 클러스터로 자동 분류한 결과입니다.
버튼을 눌러 각 그룹에 어떤 과일들이 묶였는지 확인해 보세요!

탐색기 작동 원리: 라벨(Labels)의 마법

어떤 코드가 이미지를 분류하나요?

km.labels_

학습이 끝난 `km` 모델의 `labels_` 속성에는 300개의 숫자가 들어있습니다. 예를 들어 [0, 0, 1, 2, 0, ...] 처럼 각 사진이 0번, 1번, 2번 팀 중 어디에 속하는지 기록되어 있죠. 우리가 버튼을 클릭하면, 탐색기는 이 번호표를 보고 사진들을 모아서 보여주는 것이랍니다!

클러스터별 과일을 추출하는 전체 코드

# 원하는 클러스터 번호(예: 0번)에 속한 사진 골라내기
target = np.where(km.labels_ == 0)[0]
# 10x10 격자 캔버스 만들기 (최대 100장)
fig, axes = plt.subplots(10, 10, figsize=(10, 10))
for i in range(10):
  for j in range(10):
    axes[i, j].imshow(fruits[target[i*10+j]],
        cmap='gray_r')
    axes[i, j].axis('off')
  • np.where(km.labels_ == 0): 300개의 번호표 중에서 "0번 팀"인 사진의 위치(인덱스)만 추출합니다. 마치 "0번 번호표를 가진 학생만 앞으로 나오세요!"와 같습니다.
  • plt.subplots(10, 10): 10행 10열, 총 100칸의 격자 캔버스를 준비합니다. 각 칸에 한 장씩 사진을 채우게 됩니다.
  • fruits[target[i*10+j]]: 원본 과일 데이터에서 0번 팀에 해당하는 사진을 순서대로 꺼냅니다. i*10+j는 2차원 격자를 1차원 인덱스로 변환하는 공식이에요.
  • axis('off'): 각 칸의 축 숫자를 숨깁니다. 사진만 깔끔하게 보이도록 하기 위한 설정이에요.
