⚡ 세션 8: 확률적 경사 하강법과 점진적 학습

8-1. 점진적 학습 (Incremental Learning)

데이터가 한꺼번에 준비되지 않고 조금씩 도착할 때, 기존에 학습한 내용은 유지하면서 새 데이터만 추가로 학습하는 기법입니다.

이때 사용되는 엔진이 바로 확률적 경사 하강법(SGD)입니다. 마치 산에서 내려올 때 조금씩 발을 내디디며 가장 낮은 곳(최솟값)을 찾는 것과 비슷하죠.

🎯 에포크 (Epoch)와 손실 함수

에포크 (Epoch)

전체 데이터를 한 바퀴 다 훑는 것을 1에포크라고 합니다. 너무 적으면 덜 배우고, 너무 많으면 과하게 집착합니다.

손실 함수 (Loss)

모델의 예측이 정답과 얼마나 틀렸는지 측정하는 기준입니다. 이 손실을 줄이는 방향으로 학습이 진행됩니다.

🚩 힌지 손실(Hinge Loss): 서포트 벡터 머신(SVM) 알고리즘에서 주로 쓰이는 손실 함수입니다. 분류 경계에서 멀어질수록 페널티를 주죠.
💻 코드 ① — 규제를 위한 전처리
from sklearn.preprocessing import StandardScaler

ss = StandardScaler()
ss.fit(train_input)
train_scaled = ss.transform(train_input)
핵심: 경사 하강법 알고리즘은 특성들의 스케일에 매우 민감하므로 반드시 표준화해야 합니다.

왜 여기서 shape을 계속 사용하나요?

.shape는 데이터의 (행 수, 열 수)를 알려줍니다. 예: train_input.shape → (119, 5)는 119개 샘플, 5개 특성이란 의미입니다. 데이터가 원하는 형태로 제대로 나뉘었는지 확인하기 위해 머신러닝에서 습관적으로 사용합니다.

8강에서 전처리를 왜 하나요?

경사 하강법은 특성값의 크기(스케일)에 민감합니다. 예: 길이(30cm)와 무게(500g)의 숫자 크기가 다르면 경사를 따라 내려올 때 비뚤어진 타원 경로로 움직여 학습이 느려지고 불안정해집니다. StandardScaler로 모든 특성을 평균 0, 표준편차 1로 맞추면 경사가 원형이 되어 효율적으로 최적점을 찾을 수 있습니다.

💻 코드 ② — SGDClassifier 생성
from sklearn.linear_model import SGDClassifier

sc = SGDClassifier(loss='log_loss', max_iter=10, random_state=42)
핵심: log_loss는 로지스틱 회귀를 위한 손실 함수로, 확률적으로 경사를 내려갑니다.

SGDClassifier는 왜 사용되나요?

SGDClassifier는 확률적 경사 하강법(SGD)을 이용해 분류 모델을 훈련하는 클래스입니다. 일반 로지스틱 회귀(LogisticRegression)와 달리, 데이터를 한 번에 모두 메모리에 올리지 않고도 조금씩 학습할 수 있어 거대한 데이터셋에서도 동작합니다. 또한 partial_fit()을 통해 점진적 학습(Online Learning)이 가능한 유일한 분류기입니다.

'확률적(Stochastic)'은 정확히 무엇을 의미하나요?

'확률적'이란 전체 데이터를 다 보지 않고 무작위로 뽑은 한 개(또는 소량)의 샘플만 보고 경사를 계산한다는 뜻입니다. random_state=42가 바로 이 무작위 추출의 시드값입니다. 코드에서 sc.fit()이 호출될 때 내부적으로 훈련 데이터를 무작위로 섞은 뒤 한 샘플씩 꺼내 가중치를 업데이트합니다. 전체를 다 보는 것보다 빠르지만 약간 요동치는 경로로 내려오기 때문에 '확률적'이라 합니다.

💻 코드 ③ — 전체 데이터 학습 (fit)
sc.fit(train_scaled, train_target)
print(sc.score(train_scaled, train_target))
핵심: 일반적인 fit은 데이터를 한 번 다 훑으며 최적의 가중치를 찾습니다.

.score() 점수는 무엇을 의미하나요?

sc.score()가 반환하는 값은 정확도(Accuracy)입니다. 0.0(0%) ~ 1.0(100%) 범위이며, 모델이 데이터를 맞춘 비율입니다. 최대치(1.0)는 모든 샘플을 정확히 맞춘 경우이고, 최소치(0.0)는 하나도 맞추지 못한 경우입니다. 일반적으로 훈련 점수는 0.8~0.9 이상이면 양호하며, 테스트 점수와의 차이가 작을수록 안정적인 모델입니다.

💻 코드 ④ — 점진적 학습 (partial_fit)
# 모델을 초기화하지 않고 현재 상태에서 데이터를 한 번 더 학습시킵니다.
sc.partial_fit(train_scaled, train_target)
핵심: 데이터가 너무 커서 나눠서 학습해야 할 때 아주 유용합니다.

partial_fit()은 왜 중요하고 그래프와 어떤 관련이 있나요?

partial_fit()은 모델을 초기화하지 않고 현재 가중치를 유지한 채 데이터를 한 번 더 학습시킵니다. 일반적인 fit()은 호출할 때마다 가중치가 처음부터 리셋되지만, partial_fit()은 이어서 학습합니다.

📊 그래프와의 관계: 위 '에포크 반복에 따른 가중치 진화' 그래프의 각 가로축 포인트가 바로 partial_fit() 한 번 호출에 해당합니다. 루프 안에서 for _ in range(100): sc.partial_fit(...)을 호출하면 100개의 포인트가 생기며, 가중치가 점점 안정되어가는 것을 볼 수 있습니다. 아래 실습에서 에포크를 추가할수록 그래프가 오른쪽으로 연장되는 것이 바로 이 효과입니다.

💻 코드 ⑤ — 에포크와 정확도 그래프
# 에포크가 반복될수록 훈련 세트 점수는 오르지만, 테스트 점수는 떨어질 수 있습니다.
# 이 지점이 바로 '조기 종료'가 필요한 타이밍입니다.
핵심: 과대적합이 시작되기 전 최적의 에포크 횟수를 찾는 것이 관건입니다.
💻 코드 ⑥ — 힌지 손실 (SVM 맛보기)
sc = SGDClassifier(loss='hinge', max_iter=100)
핵심: loss='hinge'를 쓰면 서포트 벡터 머신(SVM) 모델을 경사 하강법으로 훈련하게 됩니다.

힌지 손실(loss='hinge')의 한계점

힌지 손실을 사용하면 SVM 방식으로 분류 경계를 찾지만, 중요한 단점이 있습니다:

  • predict_proba() 메서드를 사용할 수 없습니다. 즉, "80% 확률로 도미입니다" 같은 확률값을 얻을 수 없고, 단순히 "도미입니다" 또는 "빙어입니다"라는 이분법적 결과만 나옵니다.
  • 반면 loss='log_loss'(로지스틱 손실)를 사용하면 predict_proba()로 각 클래스에 속할 확률을 계산할 수 있어 의사결정에 훨씬 유용합니다.
  • 의학·금융처럼 "90% 확신이 있을 때만 판정" 같은 임계값(threshold) 조정이 필요한 분야에서는 힌지 손실이 적합하지 않습니다.

따라서 실무에서는 확률이 필요한 경우 log_loss를, 마진(안전거리)을 최대화하는 명확한 경계선만 필요한 경우 hinge를 선택합니다.

SGD 학습 시각화 리포트
① 초기 가중치 상태
초기 가중치는 어떻게 부여될까요? 최초로 훈련을 시작할 때, 사이킷런의 SGDClassifier는 보통 가중치(기울기)와 절편을 0 또는 아주 작은 무작위 숫자로 시작합니다.
마치 산꼭대기에 처음 헬리콥터로 뚝 떨어졌을 때 자신이 어디 서 있는지, 어느 방향으로 내려가야 하는지 전혀 모르는 백지 상태와 같습니다.
이후 데이터를 한 스푼씩(배치) 먹어가며 오차(Loss)가 줄어드는 방향으로 가중치 막대기둥들의 길이를 조절해나갑니다.
② 에포크 반복에 따른 가중치의 진화 (Weight Evolution)
어떤 코드가 이 과정을 반복하게 할까요?

위 그래프는 에포크(Epoch)가 1에서 100까지 증가함에 따라 각 특성의 가중치(기울기)가 어떻게 변해가는지를 보여줍니다.
처음엔 0에 가깝던 값들이 요동치다가 점차 최적의 값을 찾아 수렴(안정화)하게 됩니다. 이 과정을 수행하는 핵심 코드는 다음과 같습니다. 

# 100번의 에포크 동안 반복 학습
for _ in range(100):
    # partial_fit()이 한 세대의 훈련과 가중치 업데이트를 담당!
    sc.partial_fit(train_scaled, train_target, classes=classes)
반복문 안의 partial_fit() 함수가 호출될 때마다, 산에서 한 걸음 밑으로 내려오며 가중치(기울기)를 업데이트하게 됩니다!
③ 손실 함수(Loss Function)

경사를 따라 내려오며 줄여야 할 '틀린 정도'입니다.

④ 힌지 손실 (Hinge Loss) 곡선 & 데이터 매핑

인터랙티브 분석: 예측 타겟이 되는 생선을 변경해가며 힌지 손실 산점도를 관찰해보세요.

데이터는 손실 함수 위에 어떻게 올라갈까요? 그래프의 파란 꺾은선이 바로 설정한 손실 함수(힌지 손실) 곡선이며, 그 위에 흩뿌려진 빨간 점들이 위 표에서 본 실제 훈련 데이터(생선 샘플)들입니다.

• 가로축(x축): 모델이 생선의 특성(길이, 높이, 두께 등)을 보고 계산한 '예측 점수(결정 함수 값)'입니다.
• 세로축(y축): 모델의 예측이 빗나간 정도를 수치화한 '벌점(Loss)'입니다.

👉 강의 데이터 적용법:
예를 들어 빙어 데이터 1개가 훈련에 들어오면, 현재의 가중치로 예측 점수를 계산합니다. 이 점수가 오답 구역(예: x < 1)에 위치하면, 그 오차만큼 y축 위로 올라가 '벌점'을 맞게 됩니다.
모델은 다음 번에 같은 빙어를 볼 때 벌점을 피하기 위해 가중치를 미세하게 비틉니다. 즉, 점들을 곡선의 오른쪽 아래(Loss=0이 되는 안전 지대)로 굴려 떨어뜨리는 것이 목표입니다!

8-2. SGD(확률적 경사 하강법)의 작동 원리

SGD는 마치 안개가 자욱한 산에서 발 아래의 경사만 보고 조금씩 내려오는 것과 같습니다.

  • 확률적(Stochastic): 전체 데이터가 아닌, 딱 한 개의 샘플(또는 소량)만 보고 방향을 결정합니다. 속도가 매우 빠릅니다.
  • 경사 하강(Gradient Descent): 손실 함수의 기울기(경사)가 낮아지는 쪽으로 가중치를 업데이트합니다.
  • 시간에 따른 변화: 학습 초기에는 경사가 가파르므로 크게 움직이지만, 시간이 갈수록 최적점에 가까워지며 미세하게 조정됩니다.

8-3. 손실 함수 (Loss Function)의 역할

모델이 얼마나 '나쁜가'를 측정하는 나침반입니다.

로지스틱 손실 (log_loss)

다중 분류를 위한 확률적 모델에 적합하며, 예측이 틀릴수록 벌금을 기하급수적으로 늘려 모델을 채찍질합니다.

힌지 손실 (hinge)

서포트 벡터 머신(SVM) 알고리즘을 위한 기본 손실 함수입니다. '마진'이라는 안전 거리를 확보하며 경계선을 긋습니다.

💻 코드 ⑦ — 로지스틱 손실 함수 적용하기
from sklearn.linear_model import SGDClassifier

# 손실 함수로 'log_loss'(로지스틱 손실) 지정
sc = SGDClassifier(loss='log_loss', max_iter=10, random_state=42)
sc.fit(train_scaled, train_target)
print(sc.score(train_scaled, train_target))
핵심: loss='log_loss' 파라미터를 지정하면 SGDClassifier 클래스가 로지스틱 손실 함수를 사용하도록 설정됩니다. (기본값은 'hinge'입니다.) 이를 통해 각 클래스에 모델이 부여하는 '확률'도 계산할 수 있습니다.

SGD 궁금증 해결소

Q: 에포크(Epoch)란 정확히 무엇인가요?

A: 전체 데이터를 모델이 한 번 훑고 지나가는 것을 '1 에포크'라고 합니다. 여러 번 훑을수록 데이터의 패턴을 더 깊게 이해하게 됩니다.

Q: 왜 훈련을 많이 할수록 정확도가 높아지나요?

A: 경사 하강법은 반복을 통해 '가장 오차가 적은 지점(산의 골짜기)'을 찾아가기 때문입니다. 반복이 많을수록 더 낮은 곳으로 향할 수 있습니다.

Q: 훈련 정확도는 높은데 테스트 점수가 낮아요!

A: '과대적합(Overfitting)' 신호입니다. 모델이 훈련 데이터의 사소한 내용까지 다 외워버려서 새로운 데이터에는 대처하지 못하는 상태입니다.

Q: SGD는 실무에서 왜 중요한가요?

A: 데이터가 너무 커서 한 번에 메모리에 올릴 수 없을 때, 혹은 새로운 데이터가 실시간으로 계속 들어올 때 '점진적 학습'이 필수이기 때문입니다.

Q: 최적의 에포크 수는 어떻게 찾나요?
