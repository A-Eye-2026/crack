🌊 세션 5: 거대 농어의 습격과 '회귀'의 진화

KNN의 대실수: "이웃만 믿었던 한계"

여기 아주 커다란 100cm 농어가 나타났다고 해볼까요? KNN은 당황하며 엉터리 대답을 내놓습니다.

💡 학계 노트: 외삽(Extrapolation)의 위험성

훈련 데이터의 범위를 벗어난 새로운 데이터를 예측하는 것을 '외삽'이라고 합니다. 머신러닝 모델, 특히 직선 방정식(회귀식)이나 복잡한 패턴을 학습하지 못하는 단순한 알고리즘(ex: KNN, 의사결정나무 계열)은 훈련받지 않은 미지의 영역에서 상식 이하의 예측값을 반환하는 경향이 있습니다. 현실 문제에서 모델이 예상치 못하게 무너지는 주된 원인이기도 합니다.

📍 KNN의 한계: 100cm 농어 예측 (평행선)

선형 회귀: "세상의 규칙을 직선으로"

데이터 전체를 관통하는 하나의 규칙(직선)을 찾아내면, 우리가 본 적 없는 거대 농어의 무게도 추측할 수 있습니다. 이것이 선형 회귀(Linear Regression)입니다.

모델별 예측 특성 비교
KNN 회귀

주변 이웃의 평균만 따름.
범위 밖 예측 불가능.

선형 회귀

직선의 방정식을 세움.
단순하지만 추론 가능.

다항 회귀

곡선(2차항)을 적용.
가장 자연스러운 피팅.

🎯 다항 회귀 (Polynomial Regression)의 원리

현실은 직선보다 곡선에 가까울 때가 많아요. 길이를 제곱($x^2$)한 데이터를 모델에 추가로 제공하면 직선 대신 예쁜 곡선을 그릴 수 있습니다. (차원 추가의 효과)

$$ \text{무게} = a \times \text{길이}^2 + b \times \text{길이} + c $$

선형 회귀 알고리즘 자체를 바꾸는 것이 아니라, 입력 데이터를 다항(다차원)으로 가공하여 모델에게 주는 것이 핵심입니다.

💻 코드 ① — KNN의 한계 (Extrapolation)
from sklearn.neighbors import KNeighborsRegressor
knr = KNeighborsRegressor(n_neighbors=3)
knr.fit(train_input, train_target)

# 100cm 농어의 무게를 예측해봅니다.
print(knr.predict([[100]])) # 결과는 실제보다 훨씬 작게 나올 거예요.
핵심: KNN은 훈련 데이터의 범위를 벗어나면 가장 가까운 데이터의 평균만 반복해서 출력합니다.
💻 코드 ② — 선형 회귀 모델 훈련
from sklearn.linear_model import LinearRegression

lr = LinearRegression()
lr.fit(train_input, train_target)

# 선형 회귀가 데이터의 경향성을 파악해 직선을 그었습니다.
# 이제 100cm 농어의 무게를 추론해봅니다. (KNN의 실패 극복)
print(lr.predict([[100]]))
핵심: 선형 회귀는 데이터 밖의 영역도 찾은 방정식(직선의 기울기)을 따라 계속해서 추론할 수 있습니다.
💻 코드 ③ — 계수(a)와 절편(b) 확인
print(lr.coef_, lr.intercept_)
핵심: 알고리즘이 훈련(Fit) 과정을 통해 무게 = 39.01 * 길이 - 709.01 이라는 일차방정식을 스스로 찾아낸 것입니다. 이 두 값을 파라미터(모델 파라미터)라고 부릅니다.
💻 코드 ④ — 다항 회귀를 위한 데이터 변환
import numpy as np
# 길이를 제곱한 특성을 추가하여 곡선을 만듭니다.
train_poly = np.column_stack((train_input ** 2, train_input))
test_poly = np.column_stack((test_input ** 2, test_input))
핵심: $y = ax^2 + bx + c$ 형태의 이차방정식을 풀기 위해 제곱항을 추가합니다.
💻 코드 ⑤ — 다항 회귀 모델 훈련
# 똑같은 선형 회귀 알고리즘을 사용합니다.
lr = LinearRegression()

# 차이점: 원본 길이만 넣는게 아니라, 아까 가공해둔 '제곱항'과 '기본항'을 모두 전달합니다.
lr.fit(train_poly, train_target)

# 예측할 때도 똑같이 제곱값과 기본값을 모두 넘겨주어야 합니다.
print(lr.predict([[50**2, 50]]))
핵심: 선형 모델 입장에서는 단지 특성이 2개($x_1, x_2$)인 데이터를 받아 다중선형회귀 식을 푼 것과 완전히 동일합니다. 사람이 보기에 그것이 $x_1=x^2, x_2=x$ 이기 때문에 곡선이 되는 것입니다.
💻 코드 ⑥ — 곡선 방정식 확인
print(lr.coef_, lr.intercept_)
핵심: 모델이 찾은 곡선 방정식은 무게 = $1.01 \times 길이^2 - 21.6 \times 길이 + 116.05$ 가 됩니다!

❓ 궁금할 수 있는 질문들

Q: 직선(선형)이 좋은가요, 곡선(다항)이 좋은가요?

A: 데이터의 실제 형태에 따라 다릅니다. 농어의 경우 길이가 늘어남에 따라 무게가 급격히 늘어나는 경향이 있어 곡선이 훨씬 더 정확합니다.

Q: 다항 회귀에서 차수(degree)를 무한정 높이면 어떻게 되나요?

A: 훈련 데이터의 모든 점을 다 지나는 아주 복잡한 선이 만들어집니다(과대적합). 하지만 데이터 밖의 예측 성능은 오히려 엉망이 됩니다.

Q: 절편(intercept_)값이 음수로 나오기도 하는데 괜찮나요?

A: 수학적으로는 가능하지만, 물리적으로 무게가 음수가 될 순 없겠죠. 이런 경우 모델이 아주 작은 농어 구간에서는 현실성이 떨어진다는 뜻이기도 합니다.

Q: 선형 회귀에서 '최적의 직선'은 어떻게 찾나요?

A: '최소제곱법'이라는 수학적 도구를 사용합니다. 실제 값과 예측 값의 차이(오차)를 제곱해서 모두 더한 값이 최소가 되는 직선을 찾는 방식입니다.

Q: 특성이 하나만 있는데 왜 다항 회귀가 '다중 회귀'가 아닌가요?

A: '길이'라는 하나의 특성을 가공(제곱)해서 썼기 때문입니다. 여러 개의 서로 다른 특성(길이, 높이, 두께 등)을 사용하는 것은 다음에 배울 '다중 회귀'입니다.
