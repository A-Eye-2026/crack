🎯 세션 7: 로지스틱 회귀 (분류)

🎯 시그모이드(Sigmoid) 함수의 마법

어떤 큰 숫자가 들어와도 0~1 사이로 압축해줍니다.

$$ \phi(z) = \frac{1}{1 + e^{-z}} $$

결과가 0.5보다 크면 양성(1), 작으면 음성(0)으로 판단합니다.

선형 점수 ($z$) 계산 결과 ($e^{-z}$) 분모 ($1 + e^{-z}$) 최종 확률 ($\phi(z)$)
-5.0 148.41 149.41 0.007 (0.7%)
0.0 1.00 2.00 0.500 (50.0%)
5.0 0.006 1.006 0.993 (99.3%)

7-1. 확률로 유추하기

이름은 '회귀'지만 실제로는 분류 알고리즘입니다. 선형 회귀처럼 직선을 학습한 뒤, 그 결과를 0과 1 사이의 확률로 변환해주는 시그모이드(Sigmoid) 함수를 통과시킵니다.

💻 코드 0 — 데이터 준비 및 전처리
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# 웹에서 데이터를 가져와 특성(길이, 높이, 두께, 무게 등)과 타겟(어종)으로 나눕니다.
fish = pd.read_csv('https://bit.ly/fish_csv_data')
fish_input = fish[['Weight','Length','Diagonal','Height','Width']].to_numpy()
fish_target = fish['Species'].to_numpy()

train_input, test_input, train_target, test_target = train_test_split(
    fish_input, fish_target, random_state=42
)

# 3강에서 배운 표준화 처리를 반드시 거쳐야 합니다!
ss = StandardScaler()
ss.fit(train_input)
train_scaled = ss.transform(train_input)
test_scaled = ss.transform(test_input)
핵심: 로지스틱 회귀도 선형 방정식($z = w_1x_1 + w_2x_2 + \dots + b$)을 사용하므로, 스케일링(표준화)이 필수적입니다.
💻 코드 ① — 시그모이드 함수 직접 구현
import numpy as np

# 시그모이드(로지스틱) 함수 수식입니다.
z = np.arange(-5, 5, 0.1)
phi = 1 / (1 + np.exp(-z))
핵심: 어떤 값이든 0과 1 사이로 매핑하여 '확률'처럼 해석할 수 있게 합니다.
💻 코드 ② — 이진 분류 확률 확인
# 처음 5개 샘플에 대한 예측 확률을 출력합니다.
print(lr.predict_proba(train_scaled[:5]))
핵심: predict_proba는 [음성 확률, 양성 확률] 형태로 값을 반환합니다.
💻 코드 ③ — 로지스틱 회귀로 분류하기
from sklearn.linear_model import LogisticRegression

lr = LogisticRegression()
lr.fit(train_scaled, train_target)
핵심: 이름은 회귀지만 분류 모델입니다. 직선을 찾고 시그모이드를 씌우는 방식이죠.
💻 코드 ④ — 결정 함수 (z값) 확인
# 시그모이드 함수에 들어가기 전의 원시 점수(z)입니다.
decisions = lr.decision_function(train_scaled[:5])
print(decisions)
핵심: z값이 크면 확률이 1에 가까워지고, 작으면 0에 가까워집니다.
💻 코드 ⑤ — 다중 분류와 소프트맥스
# 7종 생선을 분류할 때는 '소프트맥스' 함수를 사용합니다.
# 모든 클래스의 확률 합이 1이 되도록 만듭니다.
핵심: 이진 분류는 시그모이드, 다중 분류는 소프트맥스가 담당합니다.
💻 코드 ⑥ — 규제 조절 (C 매개변수)
# 로지스틱 회귀는 기본적으로 L2 규제가 적용됩니다.
lr = LogisticRegression(C=20, max_iter=1000)
핵심: C는 규제의 역수입니다. 값이 작을수록 규제가 강해지고, 크면 약해집니다.

7-2. 결정 경계 (Decision Boundary)

로지스틱 회귀는 데이터를 나누는 경계선을 찾습니다. 이 경계선을 기준으로 한쪽은 'A 클래스', 다른 한쪽은 'B 클래스'가 될 확률이 높아집니다.

다중 분류(7종 생선)의 경우, 각 어종마다 별도의 경계선을 만들어 가장 확률이 높은 쪽으로 최종 판정합니다.

💡 학계 노트: 어떻게 직선으로 곡선을 분류하나요?

로지스틱 회귀 모델의 결정 경계는 수학적으로 $z = 0$이 되는 '직선(또는 평면)'입니다. 하지만 우리가 6강에서 배운 특성 공학(PolynomialFeatures)을 적용하여 특성을 제곱하거나 곱해서 넘겨주면, 모델 관점에서는 직선이지만 우리 눈(원래 특성 공간)에는 둥글고 복잡한 비선형 곡선 형태의 경계선이 그려지게 됩니다.

💻 코드 ② — 로지스틱 회귀 모델 훈련
from sklearn.linear_model import LogisticRegression
# C는 규제의 역수 (낮을수록 규제가 강함)
lr = LogisticRegression(C=20, max_iter=1000)
lr.fit(train_scaled, train_target)
핵심: 다중 분류 시 내부적으로는 클래스마다 이진 분류를 수행(OvR)합니다.
💻 코드 ③ — 다중 분류 가중치(coef_)와 절편(intercept_) 확인
print(lr.coef_.shape, lr.intercept_.shape)
# (7, 5) (7,) -> 7개 어종마다 5개의 특성 가중치와 1개의 절편을 가짐
핵심: 각 생선마다 독립적인 '이진 분류기(직선의 방정식)'를 하나씩 가지고 있음을 의미합니다 (One-vs-Rest 방식).

7-3. 결정 트리로 가기 전, 하이퍼파라미터 튜닝

로지스틱 회귀도 릿지 회귀처럼 L2 규제를 기본 내장하고 있습니다. 규제를 조절하는 매개변수가 바로 C 입니다.

💡 학계 노트: 왜 $\alpha$가 아니고 C인가요?

scikit-learn의 로지스틱 회귀는 오종 알고리즘인 '서포트 벡터 머신(SVM)' 라이브러리의 규약을 따랐기 때문입니다. $\alpha$(알파)는 값이 클수록 규제가 강해지지만, C는 $1/\alpha$의 개념이므로 작을수록 규제가 강해집니다. 모델의 '탐욕도'를 억제하는 고삐라고 생각하면 쉽습니다.

7-4. 손실 함수 (Log Loss)

분류 알고리즘이 "틀렸을 때 치르는 벌금"입니다. 예측 확률이 정답에서 멀어질수록 벌점이 기하급수적으로 늘어나도록 수식을 디자인했습니다.

💡 개발자 노트: 이 로그 손실(교차 엔트로피 손실)은 인공지능과 딥러닝에서 모델을 훈련시킬 때 가장 널리 쓰이는 표준 벌점 체계입니다. 벌점이 무한대로 발산할 수 있기 때문에 모델이 "확실하게 틀린 예측"을 하는 것을 극도로 기피하게 만듭니다.

❓ 궁금할 수 있는 질문들

Q: 왜 이름이 '회귀'인가요? 분류라면서요.

A: 선형 회귀처럼 $z = w \times \text{특성} + b$ 라는 회귀 방정식(연속형 값)을 먼저 계산하기 때문입니다. 그 결과만 시그모이드를 거쳐 확률(분류)로 쓰는 것이죠. 이 과정을 확률 모델링의 통계적 용어로 '로지스틱 회귀'라고 부르게 되었습니다.

Q: 시그모이드와 소프트맥스의 차이는 명확하게 무엇인가요?

A: 시그모이드는 "합격이냐 불합격이냐(정답 2개)"를 다룰 때 쓰고, 계산 결과가 서로 독립적입니다. 소프트맥스는 "도미, 빙어, 농어 중 무엇이냐(정답 여러개)"를 다룰 때 쓰며, 모든 클래스 확률의 합이 정확히 100%(1.0)가 되도록 데이터를 압착합니다.

Q: 시그모이드 함수는 요즘 딥러닝에서도 쓰나요?

A: 출력층(마지막 결론을 낼 때)에서 이진 분류를 할 때는 여전히 100% 사용합니다. 하지만 신경망 중간 단계(은닉층)에서는 더 이상 쓰지 않습니다. $z$ 값이 너무 크거나 작으면 기울기가 0이 되어 학습이 멈추는 '기울기 소실(Gradient Vanishing)' 문제 때문인데, 이를 해결하기 위해 ReLU라는 함수가 표준으로 자리잡았습니다.

Q: 분류 결과 확률이 0.51이면 어떻게 해석해야 하나요?

A: 모델이 양성 클래스라고 판정은 하지만, 확신은 매우 낮은 상태입니다. 결정 경계(Decision Boundary, z=0인 지점)에 아주 아슬아슬하게 걸쳐 있는 데이터라는 뜻입니다.

Q: C값이 크면 모델이 어떻게 변하나요?

A: 앞서 배운 $\alpha$ 매개변수와 반대로, C가 커지면 규제가 약해집니다. 가중치(w) 값이 굉장히 커지더라도 모델이 개의치 않고 훈련 데이터에 완벽하게 맞추려 노력하게 되며, 이는 곧 복잡한 경계선을 만들어 과대적합(Overfitting)될 확률을 높입니다.
