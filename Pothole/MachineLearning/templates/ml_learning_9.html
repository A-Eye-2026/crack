🌳 세션 9: 결정 트리와 교차 검증

9-1. 결정 트리 (Decision Tree)란?

결정 트리는 마치 "스무고개 게임"과 같습니다. 데이터의 특성을 하나씩 질문하면서 점점 더 정확하게 분류해나가는 알고리즘이에요.

  • 🌿 루트 노드 (Root Node): 가장 첫 번째 질문. 가장 유용한 특성으로 데이터를 둘로 나눕니다.
  • 🍃 리프 노드 (Leaf Node): 더 이상 나눌 수 없는 최종 답변. 가장 많은 클래스가 예측값이 됩니다.
  • ✂️ 가지치기 (Pruning): 과대적합을 방지하기 위해 트리의 깊이를 제한합니다.
  • 📊 특성 중요도: 결정 트리는 어떤 특성이 분류에 가장 많은 기여를 했는지 알려줍니다.
💡 이번 세션의 데이터: 와인의 알콜 도수(Alcohol), 당도(Sugar), pH 세 가지 특성으로 레드 와인과 화이트 와인을 구분합니다!

🎯 결정 트리의 핵심 개념 3가지

🎲 불순도 (Impurity)

노드 안에 서로 다른 클래스가 얼마나 혼합되어 있는지 측정합니다. 50:50이면 최대 불순도, 한 종류만 있으면 순수합니다.

✂️ 가지치기 (Pruning)

트리가 너무 깊어지면 훈련 데이터를 외워버리는 과대적합이 발생합니다. max_depth로 깊이를 제한합니다.

📈 정보 이득 (Information Gain)

분할 전후의 불순도 차이입니다. 정보 이득이 가장 큰 특성을 기준으로 데이터를 나눕니다.

결정 트리 시각화 (max_depth=3)
훈련 점수: - 테스트 점수: -

각 노드의 색이 진할수록 한쪽 클래스 비율이 높다는 뜻입니다. 주황색은 레드 와인, 파란색은 화이트 와인입니다.

9-2. 지니 불순도 (Gini Impurity)

결정 트리가 데이터를 어떤 기준으로 나눌지 결정하는 핵심 지표입니다.

$$ \text{Gini} = 1 - \sum_{i=1}^{k} p_i^2 $$
  • 순수 노드 (gini=0): 한 클래스만 존재. 더 이상 나눌 필요가 없습니다.
  • 최대 불순도 (gini=0.5): 두 클래스가 정확히 반반. 가장 혼란스러운 상태입니다.
  • 트리는 gini 값이 가장 많이 줄어드는 방향으로 분할합니다.

9-3. 과대적합과 가지치기

가지치기 없이 트리를 성장시키면 훈련 데이터를 100% 맞히지만, 새로운 데이터에서는 오히려 성능이 떨어집니다. 이를 과대적합(Overfitting)이라 합니다.

❌ 과대적합 (가지치기 없음)

훈련 점수 99% → 테스트 점수 80%대. 훈련 데이터의 잡음까지 외워버린 상태입니다.

✅ 적절한 가지치기

훈련 점수와 테스트 점수가 비슷하게 안정됩니다. 일반화 성능이 뛰어난 상태!

9-4. 특성 중요도 (Feature Importance)

결정 트리는 분류에 어떤 특성이 가장 크게 기여했는지를 자동으로 계산해줍니다. 이를 통해 보고서에 "당도가 가장 중요한 판단 기준"이라고 근거를 제시할 수 있습니다.

🚩 결론: 결정 트리는 표준화(스케일링)가 불필요합니다! 각 특성의 단위가 달라도 상관없이 분류할 수 있습니다.

🧠 알아두면 좋은 상식

random_state=42를 왜 쓸까?

42는 더글러스 애덤스의 소설 "은하수를 여행하는 히치하이커를 위한 안내서"에서 "삶, 우주, 그리고 모든 것에 대한 궁극적인 답"이 42라는 유명한 레퍼런스에서 유래했습니다. scikit-learn 공식 문서에서도 42를 예시로 사용하며, 학계와 캐글(Kaggle) 대회에서 사실상 표준처럼 쓰이고 있습니다. 어떤 숫자를 넣든 결과의 품질에는 영향을 주지 않고, 재현성(reproducibility)만 보장합니다.

criterion: gini vs entropy

scikit-learn의 기본값은 gini이며, 실무에서도 지니 불순도가 더 많이 사용됩니다. 그 이유는 계산이 훨씬 빠르기 때문입니다. 엔트로피는 로그 연산(log₂)이 필요하지만, 지니는 제곱 연산만으로 계산됩니다. 두 지표의 결과 차이는 약 2% 미만으로, 현업에서는 기본값(gini)을 그대로 사용합니다.

의학 & 금융 분야에서의 결정 트리

결정 트리는 의학에서 진단 프로토콜로 사용됩니다. 예: "환자 열이 38°C 이상? → 기침 있음? → 흉부 X-ray 이상? → COVID-19 의심". 금융에서는 대출 심사에 활용됩니다. 단, 법률적으로 "왜 대출이 거절되었는가"를 설명해야 하므로, 결정 트리의 해석 가능성(Interpretability)이 핵심 장점입니다.

💻 코드 ⑤-1 — 로지스틱 회귀 먼저 시도하기 (왜 결정 트리가 필요했는지)
# PDF 강의에서는 먼저 로지스틱 회귀로 와인을 분류합니다
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

ss = StandardScaler()  # 로지스틱 회귀는 반드시 표준화 필요!
train_scaled = ss.fit_transform(train_input)
test_scaled = ss.transform(test_input)

lr = LogisticRegression()
lr.fit(train_scaled, train_target)
print(lr.score(train_scaled, train_target))  # 약 78%… 과소 적합!
print(lr.score(test_scaled, test_target))   # 약 78% → 80점 이하로 낮음

# 계수(가중치)와 절편 확인
print(lr.coef_, lr.intercept_)
# 알콜도수 × 0.51, 당도 × 1.67, pH × -0.69 + 절편 1.82
# → 이 값이 0보다 크면 화이트, 작으면 레드로 판정
💡 왜 결정 트리가 필요했는가: 로지스틱 회귀로는 약 78%에 그칩니다. 3개 특성으로는 직선 하나로 분리하기 어렵기 때문입니다. 그래서 PDF 강의에서 결정 트리를 도입합니다. → 아래 실험 버튼으로 직접 비교해보세요!
실험: 로지스틱 회귀 vs 결정 트리

같은 와인 데이터로 두 모델을 훈련시키고, 성능과 계수를 직접 비교합니다.

💻 코드 ⑤-2 — 표준화 없이 동일한 결과 (결정 트리의 장점)
# PDF: "표준화(표준점수) 전처리를 하지 않고 결정 트리를 사용해보자"
# 결정 트리는 특성값의 스케일에 영향을 받지 않는다!

# 표준화한 데이터로 훈련
dt_scaled = DecisionTreeClassifier(max_depth=3, random_state=42)
dt_scaled.fit(train_scaled, train_target)
print(dt_scaled.score(test_scaled, test_target))   # 84.15%

# 원본 데이터로 훈련 (표준화 없이!)
dt_raw = DecisionTreeClassifier(max_depth=3, random_state=42)
dt_raw.fit(train_input, train_target)
print(dt_raw.score(test_input, test_target))      # 84.15% → 동일!

# 왜? 결정 트리는 "sugar ≤ 1.625" 처럼 한 특성씩 비교하므로
# 다른 특성의 스케일과 무관하게 동일한 분류 경계를 만듭니다.
핵심: 이것이 결정 트리가 실무에서 사랑받는 이유 중 하나입니다. 전처리 없이도 동일한 성능! 로지스틱 회귀, SVM, KNN 등은 반드시 표준화가 필요하지만, 결정 트리는 그렇지 않습니다.
실험: 표준화 전/후 성능 비교

원본 데이터와 표준화된 데이터로 결정 트리를 각각 훈련시키고, 정확도와 특성 중요도가 정말 동일한지 검증합니다.

💻 코드 ⑤-3 — 트리를 읽고 보고서에 쓰는 법
# PDF에서 트리를 해석하는 방법 (상사에게 보고할 때!)
# 트리 읽기: 노드 → 왼쪽(True) → 오른쪽(False)

# 루트 노드: sugar ≤ 1.625  → True(왼쪽) or False(오른쪽)
# depth=1 왼쪽: sugar ≤ 4.325 → 당도가 낮은 와인을 더 분리
# depth=2 왼쪽: alcohol ≤ 11.025

# 즉, 보고서에 이렇게 적으면 됩니다:
# "당도 1.625 이상이고 4.325 이하인 와인 중에서
#  알콜 도수가 11.025 이하이면 레드 와인으로 판정됩니다."

# 표준화된 값(-0.239 등)이 아닌
# 원본 값(1.625 등)으로 해석 가능 → 보고서에 바로 사용!
💡실무 TIP: 표준화를 하면 노드의 기준값이 -0.239 같은 음수가 됩니다. 상사에게 "당도 -0.239 이하가 레드 와인"이라고 보고할 수 없죠! 그래서 결정 트리에서는 원본 데이터를 그대로 사용합니다.
💻 코드 ⑤-4 — 불균형 트리 (min_impurity_decrease 활용)
# max_depth 대신 min_impurity_decrease로 가지치기
# 좌우가 균일하지 않은 트리가 만들어집니다

dt = DecisionTreeClassifier(min_impurity_decrease=0.0005, random_state=42)
dt.fit(train_input, train_target)
print(dt.score(train_input, train_target))  # ~89% (max_depth=3의 85%보다 높음)
print(dt.score(test_input, test_target))   # ~86% (과대적합 아직 적음)

# min_impurity_decrease: 불순도가 이 값 이상 줄어들 때만 분할
# 값이 클수록 → 짧은 트리 (강한 가지치기)
# 값이 작을수록 → 복잡한 트리 (약한 가지치기)
핵심: 불균형 트리는 데이터의 분포에 따라 한쪽이 더 깊어질 수 있습니다. → 아래 실험 버튼으로 모양을 확인하세요!

9-5. 검증 세트와 교차 검증 (Cross Validation)

❓ 깊이 파보기: 검증 세트와 교차 검증
1. 왜 갑자기 '검증 세트(Validation Set)'가 등장했나요? 기존(훈련/테스트)과 뭐가 다르죠?

A: 이전까지는 훈련 세트로 공부하고 테스트 세트로 시험을 쳤습니다. 그런데 모델 성능을 높이려고 하이퍼파라미터(트리 깊이 등)를 계속 튜닝하면서 테스트 세트에 맞춰서 수정하다 보면, 결국 테스트 세트의 정답지마저 유출(Data Leakage)되어 진짜 실력을 알 수 없게 됩니다. 그래서 튜닝용 모의고사인 '검증 세트'를 훈련 세트 안에서 따로 떼어내어 사용하는 것입니다.

2. CV와 Fold는 무엇이고, 에포크(Epoch)와는 어떻게 다른가요?

A: 에포크(8강)는 '같은 데이터'를 반복해서 여러 번 학습하는 횟수입니다(SGD). 반면 Fold(폴드)는 데이터를 케이크 조각처럼 K개로 나누는 '데이터의 덩어리'를 뜻하고, CV(Cross Validation, 교차 검증)는 이 K개의 폴드 중 1개를 번갈아가며 검증용으로 쓰는 '평가 기법'입니다. 에포크는 모델을 학습시키는 방법, CV는 성능을 평가하는 방법으로 목적이 완전히 다릅니다. 당연히 딥러닝을 포함한 앞으로의 모든 모델 평가에서 CV 기법을 쓸 수 있습니다!

"시험 문제를 한 종류만 풀어보고 실력을 판단하면 공정할까?" 교차 검증은 데이터를 K등분(Fold)하여 돌아가며 검증함으로써 더 정확한 성능 평가를 합니다.

  • K-Fold: 데이터를 K등분 → 한 조각을 테스트, 나머지로 훈련 → K번 반복
  • 장점: 훈련 데이터를 최대한 활용할 수 있고, 특정 데이터 배치에 따른 점수 편차를 줄여 평가가 훨씬 안정적입니다.
  • 한계: K번 훈련해야 하므로 시간이 K배 소요됩니다.
💻 코드 ⑥ — 5-Fold 교차 검증
from sklearn.model_selection import cross_validate

scores = cross_validate(dt, train_input, train_target, cv=5, return_train_score=True)
print(np.mean(scores['test_score']))  # 평균 검증 점수
핵심: cv=5는 5등분을 의미합니다. 검증 점수의 평균이 최종 성능 지표로 사용됩니다.
실습: 5-Fold 교차 검증 실행

데이터를 5등분하여 교차 검증을 수행합니다. 결과로 각 폴드의 정확도가 막대 그래프로 표시됩니다.

9-6. 하이퍼파라미터 (Hyperparameter) 튜닝이란?

❓ 깊이 파보기: 하이퍼파라미터
1. 하이퍼파라미터 튜닝은 왜 쓰나요?

A: 모델이 스스로 학습할 수 없는 설정값들을 '하이퍼파라미터(Hyperparameter)'라고 부릅니다. 예를 들어 트리의 최대 깊이(max_depth)나 가지치기 최소 조건 같은 값들입니다. 이런 값들을 사람이 직접 조금씩 바꿔가면서 과대적합도 피하고 성능도 제일 높게 나오는 황금 비율을 찾는 과정을 '튜닝'이라고 합니다.

실험: 하이퍼파라미터 직접 튜닝해보기

아래 슬라이더를 움직여 파라미터 값을 조절해보세요. 설정값에 따라 훈련/검증 점수가 어떻게 변하고 트리의 모양은 어떻게 달라지는지 우측에서 즉시 확인할 수 있습니다.

값을 키울수록 트리가 깊어지며 훈련 세트에 맞춰집니다(과대적합 위험).

가지치기를 할 때 필요한 최소 개선량입니다. 값이 크면 가지치기가 듬성듬성 일어납니다.

노드를 또 분할하기 위해 필요한 최소 데이터 개수입니다.

좌측 슬라이더를 조작하면 이 곳에 실시간 성능 점수와 시각화가 나타납니다.

9-7. 자동화: 그리드 서치 (Grid Search)

위 실험처럼 사람이 일일이 슬라이더를 돌리기엔 경우의 수가 너무 많습니다. 하이퍼파라미터의 최적 조합을 찾는 자동 탐색 기법입니다. 사전에 정한 후보 값들의 모든 조합을 시도하고, 교차 검증을 통해 가장 성능이 좋은 조합을 선정합니다.

max_depth

트리의 최대 깊이. 깊을수록 복잡해지지만 과대적합 위험 증가

min_impurity_decrease

분할 시 최소 불순도 감소량. 미세한 개선은 무시

min_samples_split

노드 분할에 필요한 최소 샘플 수

⏱️ 그리드 서치 시간복잡도 계산 (PDF에서 강조!)

만약 3개 하이퍼파라미터를 다음과 같이 설정하면:

  • min_impurity_decrease: 9개 값 (0.0001~0.001)
  • max_depth: 15개 값 (5~20)
  • min_samples_split: 10개 값 (2~100)

조합 수 = 9 × 15 × 10 = 1,350 → 5-Fold 교차 검증 시 총 6,750번 모델을 훈련합니다! 이것이 랜덤 서치가 필요한 이유입니다.

📋 DecisionTreeClassifier 주요 매개변수 총정리
criterion 분할 측정 기준 (default: gini)
splitter 분할 전략 (default: best)
max_depth 트리의 최대 깊이
min_samples_split 분할 필요 최소 샘플 수 (default: 2)
min_samples_leaf 리프 노드 최소 샘플 수 (default: 1)
max_features 분할에 사용할 특성의 최대 수
max_leaf_nodes 리프 노드 최대 수
min_impurity_decrease 분할 시 최소 불순도 감소량
💻 코드 ⑦ — 그리드 서치
from sklearn.model_selection import GridSearchCV

params = {
    'min_impurity_decrease': np.arange(0.0001, 0.001, 0.0001),
    'max_depth': range(3, 15),
    'min_samples_split': range(2, 30, 5)
}
gs = GridSearchCV(DecisionTreeClassifier(random_state=42), params, n_jobs=-1, cv=5)
gs.fit(train_input, train_target)
print(gs.best_params_) # 최적의 하이퍼파라미터 조합
핵심: n_jobs=-1은 CPU 모든 코어를 사용해 병렬 처리합니다. 수백~수천 개 조합을 자동으로 탐색합니다.
실습: 그리드 서치 실행

9×12×6 = 648가지 조합을 각각 5-Fold 교차 검증으로 테스트합니다. (약 30초~1분 소요)

9-8. 랜덤 서치 (Randomized Search)

그리드 서치는 모든 조합을 시도하지만, 후보 범위가 넓으면 시간이 기하급수적으로 증가합니다. 랜덤 서치는 확률 분포에서 무작위로 샘플링하여 효율적으로 최적값을 찾습니다.

  • 균등 분포 (Uniform): 실수값 범위에서 고르게 샘플링
  • 정수 균등 분포 (Randint): 정수값 범위에서 고르게 샘플링
  • n_iter: 탐색 횟수를 제한하여 시간을 통제합니다
💻 코드 ⑧ — 랜덤 서치
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import uniform, randint

params = {
    'min_impurity_decrease': uniform(0.0001, 0.001),
    'max_depth': randint(3, 30),
    'min_samples_split': randint(2, 25),
    'min_samples_leaf': randint(1, 25)
}

rs = RandomizedSearchCV(dt, params, n_iter=100, n_jobs=-1, cv=5)
rs.fit(train_input, train_target)
핵심: n_iter=100은 100번만 무작위 조합을 시도합니다. 그리드 서치보다 훨씬 빠르면서도 좋은 결과를 얻을 수 있습니다.
실습: 랜덤 서치 실행

확률 분포에서 100회 무작위 샘플링으로 최적 조합을 탐색합니다.

📗 세션 9 용어 사전
결정 트리 (Decision Tree): if/else 질문을 반복하여 데이터를 분류하는 알고리즘
지니 불순도 (Gini Impurity): 노드에 서로 다른 클래스가 얼마나 섞여있는지 측정하는 기준
가지치기 (Pruning): max_depth 등을 사용해 트리의 과도한 성장을 방지하는 기법
교차 검증 (Cross Validation): 데이터를 K등분하여 번갈아 사용함으로써 안정적 성능 평가
그리드 서치 (Grid Search): 하이퍼파라미터의 모든 조합을 완전탐색하는 기법
랜덤 서치 (Random Search): 확률 분포에서 무작위 샘플링으로 최적값을 효율적으로 탐색

❓ 궁금할 수 있는 질문들

Q: 결정 트리는 분류만 가능한가요?

A: 아닙니다. DecisionTreeRegressor를 사용하면 회귀 문제(가격 예측 등)에도 사용할 수 있습니다. 리프 노드에서 클래스 대신 평균값을 반환합니다.

Q: 로지스틱 회귀보다 결정 트리가 더 좋은 건가요?

A: 경우에 따라 다릅니다. 결정 트리는 비선형 경계를 자연스럽게 만들 수 있고, 결과를 해석하기 쉽습니다. 반면 로지스틱 회귀는 확률값을 제공하고 대규모 데이터에서 더 안정적입니다.

Q: 교차 검증은 왜 단순 train/test 분할보다 좋은가요?

A: 단순 분할은 어떤 데이터가 테스트 세트에 들어가느냐에 따라 결과가 크게 달라질 수 있습니다. 교차 검증은 모든 데이터를 균등하게 사용하므로 운이 아닌 진짜 실력을 측정합니다.

Q: 그리드 서치 vs 랜덤 서치, 어떤 것을 써야 하나요?

A: 후보값이 적고 정확한 탐색이 필요하면 그리드 서치, 탐색 범위가 넓거나 시간이 부족하면 랜덤 서치가 효율적입니다. 실무에서는 먼저 랜덤 서치로 대략적인 범위를 좁힌 후, 그리드 서치로 미세 조정하는 방법을 많이 씁니다.

Q: 결정 트리에서 표준화(스케일링)는 필요한가요?

A: 필요 없습니다! 결정 트리는 각 특성을 독립적으로 비교(≤ / > 기준)하기 때문에, 특성의 스케일이 달라도 성능에 영향을 주지 않습니다. 이는 로지스틱 회귀나 SVM과의 큰 차이점입니다.

Q: min_samples_split과 min_samples_leaf의 차이는?

A: min_samples_split은 노드를 분할하기 위한 최소 샘플 수이고, min_samples_leaf는 리프 노드에 남아있어야 할 최소 샘플 수입니다. 둘 다 과대적합을 방지하는 역할을 합니다.
