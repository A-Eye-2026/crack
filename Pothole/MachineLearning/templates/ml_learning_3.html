📏 세션 3: 데이터 전처리 (Data Preprocessing)

3-1. 수치의 단위가 다르면?

길이는 보통 10~40cm인데, 무게는 200~1000g까지 올라가죠. 이 상태로 KNN을 쓰면 어떻게 될까요?

  • ⚖️ 스케일(Scale): 무게의 숫자가 훨씬 크기 때문에, 알고리즘은 거리를 계산할 때 무게에만 엄청난 점수를 줍니다.
  • 🪄 표준화(Standardization): 모든 특성을 평균 0, 표준편차 1로 맞춰서 공평하게 비교하는 과정입니다.

🎯 표준 점수(Z-Score) 계산법

각 데이터가 평균에서 얼마나 떨어져 있는지를 표준편차 단위로 나타냅니다.

$$ z = \frac{\text{값} - \text{평균}}{\text{표준편차}} $$

💡 학계 노트: 통계량 누설(Data Leakage) 방지

수동으로 계산할 때는 반드시 훈련 세트만의 평균과 표준편차를 구해야 하며, 테스트 세트의 데이터를 오염(포함)시켜서는 안 됩니다. 나중에 테스트를 하거나 새로운 생선이 왔을 때도, 오직 '훈련 세트에서 구했던 기존의 평균과 표준편차'를 사용하여 변환(transform)해야 모델이 실전에서도 공정하게 평가됩니다.

💻 코드 ① — 전처리 전의 문제점
import numpy as np
from sklearn.model_selection import train_test_split

# 도미와 빙어 데이터를 합치고 훈련/테스트 세트로 나눕니다.
train_input, test_input, train_target, test_target = train_test_split(
    fish_data, fish_target, stratify=fish_target, random_state=42
)

# 전처리 없이 KNN을 훈련하면?
from sklearn.neighbors import KNeighborsClassifier
kn = KNeighborsClassifier()
kn.fit(train_input, train_target)
print(kn.score(test_input, test_target)) # 정확도가 낮을 수 있습니다.
핵심: 길이(cm)와 무게(g)의 스케일이 너무 달라 거리가 왜곡됩니다.
💻 코드 ② — 평균과 표준편차 계산
mean = np.mean(train_input, axis=0)
std = np.std(train_input, axis=0)

print(mean, std)
핵심: 각 특성(길이, 무게)별로 평균과 표준편차를 구합니다.
💻 코드 ③ — 표준 점수(Z-Score)로 변환
# (데이터 - 평균) / 표준편차
train_scaled = (train_input - mean) / std
핵심: 모든 데이터를 평균이 0, 표준편차가 1인 범위로 재조정합니다.
💻 코드 ④ — 브로드캐스팅의 마법
# 넘파이는 배열 전체에 행렬 연산을 한꺼번에 적용합니다.
# 이를 '브로드캐스팅'이라고 부릅니다.
핵심: 반복문 없이 수식 한 줄로 모든 샘플을 전처리할 수 있습니다.
💻 코드 ⑤ — 새로운 데이터도 똑같이 전처리!
new_fish = ([25, 150] - mean) / std
prediction = kn.predict([new_fish])
핵심: 예측할 데이터도 반드시 '훈련 세트의 평균과 표준편차'로 변환해야 합니다.
💻 코드 ⑥ — StandardScaler 사용하기
from sklearn.preprocessing import StandardScaler

ss = StandardScaler()
ss.fit(train_input)
train_scaled = ss.transform(train_input)
핵심: 사이킷런의 StandardScaler를 쓰면 전처리를 훨씬 편하고 안전하게 할 수 있습니다.
📗 용어 사전
표준 점수 (Z-Score): 평균과의 차이를 표준편차로 나눈 값
브로드캐스팅: 넘파이 배열에서 모든 원소에 동일한 연산을 한꺼번에 적용하는 기능

🧪 실습 — 표준화의 마법 확인하기

새로운 생선 데이터를 설정하고 버튼을 누르면 표준화 전(Raw)과 후(Scaled)의 분포 변화를 분석합니다.

❓ 궁금할 수 있는 질문들

Q: 왜 테스트 세트를 전처리할 때 fit()을 다시 하면 안 되나요?

A: 모델은 훈련 세트의 '기준(평균/표준편차)'으로 학습되었기 때문입니다. 테스트 세트만 따로 fit하면 기준이 틀려져서 예측이 엉망이 됩니다.

Q: 모든 데이터에 전처리가 필요한가요?

A: 거리 기반 알고리즘(KNN, SVM, K-Means 등)은 거의 필수입니다. 반면 결정 트리(Decision Tree) 같은 방식은 스케일에 영향을 받지 않아 필요 없습니다.

Q: 표준화 말고 다른 전처리 방법이 있나요?

A: 최소-최대 정규화(Min-Max Scaling)가 대표적입니다. 데이터를 0~1 사이로 압축하는 방식이지만, 이상치(Outlier)에 민감하다는 단점이 있습니다.

Q: 평균과 표준편차를 직접 계산하는 것과 StandardScaler를 쓰는 것의 차이는?

A: 결과는 같지만, StandardScaler를 쓰면 훈련 세트의 통계치를 객체에 저장해두었다가 나중에 새로운 데이터나 테스트 세트에 그대로 적용하기 훨씬 편리하고 실수를 줄여줍니다.

Q: 데이터가 이미 단위가 같은데도 전처리를 해야 하나요?

A: 특성들 사이의 수치 범위(Scale)가 비슷하다면 생략할 수도 있습니다. 하지만 전처리를 해서 손해 볼 일은 거의 없기 때문에 습관적으로 해주는 것이 좋습니다.
