✂️ 세션 2: 훈련 세트와 테스트 세트

2-1. 왜 데이터를 나눠야 할까요?

훈련에 사용한 데이터로 성능을 점수를 매기는 것은 마치 "수능 문제를 미리 다 풀어보고 수능을 보는 것"과 같습니다. 실력을 제대로 알 수 없겠죠?

  • 🎯 훈련 세트(Train Set): 모델이 공부하는 데 사용하는 데이터
  • 📋 테스트 세트(Test Set): 모델이 얼마나 잘 배웠는지 평가하는 데이터 (실전 시험)

🎯 샘플링 편향 (Sampling Bias) 주의보!

데이터를 섞지 않고 그냥 위에서부터 자르면 어떻게 될까요? 위 차트처럼 앞쪽 35개(도미)가 모두 훈련 세트로, 뒤쪽 14개(빙어)가 모두 테스트 세트로 배정되는 사태가 벌어집니다.

결과적으로 모델은 빙어를 한 번도 못 봤으니 빙어를 맞힐 수 없게 되죠. 이것이 바로 샘플링 편향입니다. 따라서 반드시 데이터를 섞어주어야 합니다.

✨ 데이터 분할의 두 가지 방법
❌ 단순 순차 분할 (Bad)

데이터 순서대로 나눔. 특정 종류가 한쪽에 쏠릴 위험이 매우 큼 (샘플링 편향).

✅ 층화 추출 (Stratified Split)

도미와 빙어의 비율을 원본과 똑같이 맞춰서 골고루 나눠줌. 안정적인 평가 가능.

왜 층화 추출인가요? 데이터셋이 작거나 클래스 비율이 불균형할 때, 무작위로만 나누면 특정 클래스가 테스트 세트에 아예 포함되지 않을 수 있습니다. 이를 방지하여 평가의 객관성을 확보합니다.

💻 코드 ① — 넘파이 배열 만들기
import numpy as np

# 데이터를 파이썬 리스트에서 넘파이 배열로 바꿉니다.
fish_data = np.array(fish_data)
fish_target = np.array(fish_target)
핵심: 넘파이 배열은 슬라이싱과 인덱싱이 강력하여 데이터 처리에 필수적입니다.
💻 코드 ② — 부적절한 데이터 분할 (Bad Case)
# 0~34번까지는 훈련, 35~48번까지는 테스트용으로 나눕니다.
train_input = fish_data[:35]
test_input = fish_data[35:]
핵심: 데이터가 섞여있지 않다면 훈련 세트에는 도미만, 테스트 세트에는 빙어만 담길 수 있습니다.
💻 코드 ③ — 데이터 섞기 (Suffle)
index = np.arange(49)
np.random.shuffle(index) # 인덱스를 마구 섞습니다.

train_input = fish_data[index[:35]]
train_target = fish_target[index[:35]]
핵심: 데이터를 섞어야 훈련/테스트 세트에 모든 종류의 생선이 골고루 포함됩니다.
💻 코드 ④ — 사이킷런으로 스마트하게 나누기
from sklearn.model_selection import train_test_split

train_input, test_input, train_target, test_target = train_test_split(
    fish_data, fish_target, test_size=0.25, random_state=42, stratify=fish_target
)
핵심: train_test_split()은 섞기(Shuffle)와 나누기를 1줄로 처리해줍니다. 또한, stratify 파라미터를 넘겨주면 내부적으로 클래스 비율을 파악해 원본과 동일하게 훈련/테스트 세트를 쪼개줍니다.

💡 학계 노트: 왜 하필 random_state=42 인가요?

은하수를 여행하는 히치하이커를 위한 안내서라는 소설에서 '삶, 우주, 그리고 모든 것에 대한 궁극적인 해답'이 42로 등장합니다. 이를 오마주하여 컴퓨터 과학자들과 데이터 사이언티스트들이 일종의 밈(Meme)처럼 시드 고정값으로 42를 즐겨 사용하게 되었습니다. 참고로 재현성(Reproducibility)은 동료 연구자나 다른 사람들이 동일한 환경에서 동일한 결과를 얻도록 보장해야 하므로 매우 중요한 원칙입니다.

💻 코드 ⑤ — 샘플링 편향 확인하기
# 테스트 세트의 정답 비율을 확인해봅니다.
print(test_target)
# 결과가 [0, 0, 1, 0...] 처럼 한쪽으로 쏠렸다면 편향된 것입니다.
핵심: 정답 데이터의 비율이 원본과 다르면 모델의 성능 평가를 믿을 수 없게 됩니다.
💻 코드 ⑦ — 모델 훈련 및 정확도 확인
kn.fit(train_input, train_target)
score = kn.score(test_input, test_target)
print("테스트 세트 정확도:", score)
핵심: 훈련 세트로 공부한 모델이 처음 본 테스트 세트에서 얼마나 잘 맞히는지 확인합니다.
💻 코드 ⑧ — 새로운 데이터 예측하기
new_fish = [[25, 150]] # 길이 25, 무게 150인 새로운 생선
prediction = kn.predict(new_fish)
print("예측 결과:", "도미" if prediction[0] == 1 else "빙어")
핵심: 훈련과 검증이 끝난 모델을 사용하여 실제 문제를 해결하는 단계입니다.
📗 세션 2 용어 사전
샘플링 편향: 데이터가 한쪽으로 쏠려서 잘못된 패턴을 배우는 현상
Stratify (층화): 정답 비율을 유지하며 데이터를 섞는 필수 옵션

❓ 궁금할 수 있는 질문들

Q: 왜 데이터를 섞지 않으면 안 되나요?

A: 데이터가 순서대로 쌓여있는 경우, 훈련 세트에는 특정 종류만 들어가고 테스트 세트에는 다른 종류만 들어갈 수 있기 때문입니다. 이를 '샘플링 편향'이라고 합니다.

Q: random_state를 42로 하는 특별한 이유가 있나요?

A: 특별한 의미는 없지만, 코드를 다시 실행해도 항상 같은 결과를 얻기 위해 숫자를 고정하는 것입니다. 관습적으로 42를 많이 씁니다.

Q: 층화 추출(Stratify)은 언제 쓰나요?

A: 분류 문제에서 정답 간의 비율이 깨지면 안 될 때 필수입니다. 특히 데이터가 적거나 클래스 간 불균형이 심할 때 더욱 중요합니다.

Q: 테스트 비율을 50% 이상으로 높이면 어떻게 되나요?

A: 모델이 공부할 데이터(훈련 세트)가 너무 적어져서 충분한 패턴을 익히지 못하게 됩니다. 보통 20~30% 정도가 가장 적당합니다.

Q: 층화 추출을 안 하면 모델 성능이 떨어지나요?

A: 모델 자체의 잠재 성능이 떨어지는 것은 아니지만, 평가 데이터가 한쪽으로 쏠리면 모델의 진짜 실력을 측정할 수 없게 되어 결과적으로 신뢰도가 떨어집니다.
