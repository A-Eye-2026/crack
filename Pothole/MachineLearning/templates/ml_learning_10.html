🌲 세션 10: 트리의 앙상블 (Ensemble Learning)

집단 지성의 힘: 앙상블 학습

앙상블(Ensemble) 학습은 더 좋은 예측 결과를 만들기 위해 여러 개의 모델을 훈련하는 머신러닝 알고리즘입니다. 하나의 뛰어난 전문가보다 여러 명의 일반인이 내린 결정이 더 나을 수 있다는 "집단 지성"을 머신러닝에 적용한 것입니다.

대표적인 앙상블 모델:

  • 랜덤 포레스트 (Random Forest) : 결정 트리를 랜덤하게 여러 개 만들어 숲(Forest)을 이루게 한 뒤, 트리들의 투표로 결과를 결정합니다. 사이킷런에서는 기본적으로 100개의 트리를 만들어 반복 학습합니다.
  • 엑스트라 트리 (Extra Trees) : 랜덤 포레스트와 유사하지만, 트리의 노드를 분할할 때 무작위로 분할해 속도를 극대화합니다. 역시 기본값은 100개의 트리입니다.
  • 그레이디언트 부스팅 (Gradient Boosting) : 이전 트리의 오차를 보완하는 방식으로 새로운 트리를 연속해서 추가합니다. 기본적으로 100번의 반복을 통해 모델을 정교하게 다듬습니다.
Mastery Note: 왜 앙상블인가?
  • 과대적합 방지: 개별 트리가 데이터에 너무 의존하는 것을 방지합니다.
  • 일반화 성능: "평균"의 힘을 빌려 새로운 데이터에도 유연하게 대처합니다.
  • 실무 표준: 정형 데이터 분석에서 가장 높은 성능을 자랑하는 기법입니다.
훈련과 테스트 세트 모두 100%가 나올 수 있나요?
네, 그렇습니다! 랜덤 포레스트는 개별 트리를 최대한 깊게 성장시켜 훈련 데이터를 완벽히 학습(100%)합니다. 동시에, 100개의 서로 다른 트리가 투표를 하는 앙상블 효과 덕분에 테스트 데이터에서도 완벽한 결정 경계를 찾아낼 수 있습니다.
앙상블 기법 성능 비교

동일한 화이트 와인 분류 문제에 대해 앙상블 기법 간의 훈련 세트와 테스트 세트의 정확도를 비교합니다.

데이터 분석 중...

훈련 세트 100%의 진실

훈련 데이터에서 100%가 나왔다는 것은, 모델이 이미 본 문제는 완벽하게 외웠다는 뜻입니다. 앙상블은 각 나무들이 데이터를 잘게 쪼개어 학습하기 때문에, 훈련용 데이터에 대해서는 한 치의 오차도 없는 완벽한 실력을 뽐내게 됩니다.

테스트 점수가 낮은 이유

테스트 점수가 100%보다 낮다면, 이는 "한 번도 본 적 없는 데이터" 중에 모델이 분류하기에 매우 까다롭거나 애매한 경계에 있는 샘플이 섞여 있었다는 의미입니다. 즉, 제대로 분류를 못한 것보다는 현실 데이터의 복잡성을 모델이 완전히 담아내지 못했을 때 발생하는 자연스러운 오차입니다.

퍼센트(%) 수치의 진짜 의미

정확도 85%라는 수치를 단순히 '실력'으로만 보지 마세요. 앙상블 기법에서 이 숫자는 다음을 의미합니다:

  • 확신의 정도: 100개의 나무 중 85개가 "이것은 화이트 와인이다!"라고 강력하게 외쳤을 때 결과적으로 그 판단이 맞았을 확률입니다.
  • 데이터의 명확성: 100%에 가까울수록 데이터의 특성이 뚜렷하여 구분하기 쉬운 것이고, 낮을수록 노이즈(오차)가 많거나 두 부류가 서로 닮아 있어 "구분하기 애매한 경계선"에 데이터가 많다는 뜻입니다.
💻 핵심 파이썬 코드 — 앙상블 모델 정의 및 훈련
앙상블 모델의 3대장 구현하기
# 1. 랜덤 포레스트: 100개의 나무를 심어 숲을 만들고 '투표'로 결정합니다.
# (가장 안정적이고 표준적인 방법입니다!)
rf = RandomForestClassifier(n_jobs=-1, random_state=42)
rf.fit(train_input, train_target)

# 2. 엑스트라 트리: 나무를 더 대충(?) 막 만들어서 연산 속도를 엄청나게 올립니다.
# (무작위성이 매우 높지만, 때로는 랜덤 포레스트보다 더 똑똑할 때도 있어요.)
et = ExtraTreesClassifier(n_jobs=-1, random_state=42)
et.fit(train_input, train_target)

# 3. 그레이디언트 부스팅: 약한 모델부터 시작해서 오답 노트를 쓰듯 보강해가며 성장합니다.
# (속도는 조금 느리지만, 끈기 있게 학습하여 가장 높은 정확도를 노립니다.)
gb = GradientBoostingClassifier(random_state=42)
gb.fit(train_input, train_target)
n_jobs=-1 (병렬 처리)

혼자서 나무 100개를 심으려면 시간이 오래 걸리겠죠? 이 설정은 컴퓨터에 있는 모든 일꾼(CPU 코어)을 총동원해서 나무들을 동시에 심으라고 명령하는 것입니다. 학습 속도가 수십 배 빨라집니다!

fit() 메서드 (집중 학습)

준비한 훈련 데이터를 모델에게 건네주며 "자, 이제 공부해!"라고 시키는 단계입니다. 앙상블 모델은 이 과정에서 데이터를 무작위로 섞어가며 수많은 나무(기본 100개)를 만들어냅니다.

random_state (결과 고정)

앙상블은 무작위성이 강해서 실행할 때마다 결과가 조금씩 바뀔 수 있습니다. 이 숫자를 42로 고정하면, 누가 언제 실행하더라도 항상 똑같은 무작위 나무들을 만들어내어 실험 결과를 일정하게 유지합니다.

랜덤 포레스트는 각 트리가 무작위 데이터를 사용하므로, 한 트리가 틀려도 나머지 나무들이 정답을 맞히며 보완합니다.

데이터 과학자의 돋보기: 숲속의 나무들 (Sample Trees)

코드 ①(랜덤 포레스트)가 실제로 만드는 100개의 나무 중 2개를 꺼내 보았습니다. 보시는 것처럼 각 나무는 서로 다른 질문(조건)을 던지며 데이터를 분류합니다.

이 서로 다른 나무들이 투표(다수결)를 통해 최종 결정을 내리기 때문에, 한두 개의 나무가 틀리더라도 전체 결과는 매우 정확해집니다.

알고리즘별 '특성 중요도' 한눈에 비교

코드 ①, ②, ③, ④가 데이터를 바라보는 시각의 차이를 확인해보세요. 모든 모델이 '당도(sugar)'를 중요하게 보지만, 세부적인 가중치는 조금씩 다릅니다.

코드 연동 안내:
  • Random Forest: 코드 ①의 결과. 데이터를 무작위로 샘플링하여 중요도를 계산.
  • Extra Trees: 코드 ②의 결과. 분할까지 무작위로 하여 계산 속도가 가장 빠름.
  • Boosting 계열: 코드 ③, ④의 결과. 오차를 줄이는 과정에서 높은 기여도를 가진 특성을 선별.

궁금할 수 있는 질문들

Q: 트리가 100개나 되는데 너무 느리지 않을까요?

A: 랜덤 포레스트는 각 트리가 독립적으로 만들어지기 때문에 CPU의 여러 핵심(코어)을 동시에 사용해 병렬 처리가 가능합니다. 덕분에 매우 빠릅니다!

Q: 왜 이름 앞에 '랜덤'이 붙나요?

A: 각 트리를 만들 때 전체 데이터를 쓰는 게 아니라 일부분을 무작위로 뽑아 쓰고(부트스트랩 샘플), 노드를 나눌 때도 무작위로 몇 개의 특성만 골라 사용하기 때문입니다.

Q: 훈련 세트가 100%인데 과대적합 아닌가요?

A: 개별 트리는 과대적합일지 모르나, 수많은 트리의 결과를 뭉쳐서 판단하면 오히려 과대적합을 억제하는 효과(규제)가 발생합니다. 이것이 앙상블의 마법입니다.

Q: 랜덤 포레스트와 엑스트라 트리의 결정적인 차이는?

A: 랜덤 포레스트는 최적의 분할 지점을 찾지만, 엑스트라 트리는 분할 지점까지 '랜덤'하게 정합니다. 덕분에 엑스트라 트리가 더 빠르고 과대적합 방지 효과가 큰 경우도 많습니다.

Q: 그레이디언트 부스팅은 언제 사용하는 것이 좋나요?

A: 랜덤 포레스트보다 더 정교한 모델이 필요할 때 사용합니다. 오차를 순차적으로 줄여나가기 때문에 최고 수준의 정확도를 내는 경우가 많지만, 훈련 시간은 조금 더 걸립니다.

알아두면 좋을 팁

1. OOB(Out-of-Bag) 점수

랜덤 포레스트는 무작위로 데이터를 뽑을 때 선택되지 않은 남은 데이터(OOB)를 이용해 마치 테스트 세트처럼 자체 검증을 할 수 있습니다.

2. 실무에서의 활용

정형 데이터(표 형태)에서는 앙상블 기법(특히 XGBoost나 LightGBM 같은 부스팅 계열)은 현재 머신러닝의 끝판왕으로 불립니다.

3. 특성 중요도의 함정

특성 중요도는 모델의 관점일 뿐, 무조건적인 인과관계를 의미하지는 않습니다. 다른 모델과 비교하면 결과가 조금씩 다를 수 있음을 주의하세요.
