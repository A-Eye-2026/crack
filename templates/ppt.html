Werkzeug의 PBKDF2 사용. Django, Flask의 기본값으로 채택된 지극히 검증된 보안 표준입니다. 60만 번의 반복 연산으로 무차별 공격을 무력화합니다.

해외 사례를 벤치마킹한 CRACK의 B2G2B 비전

1. 법적 책임

공식적인 영조물 배상책임 근거로 쓰일 수 있도록 지자체의 법적 기록망에 직접 전달되어야 함

B2G 파이프라인

향후 Open API를 통해 지자체망 다이렉트 Payload 연동을 추진하여 공식 행정망 편입

2. 행정망 연동

폐쇄적 인트라넷 접근의 어려움과 복잡한 작업 지시 시스템(워크플로우) 통합 비용 발생

표준 JSON & Admin

시스템 이식성을 최우선한 포맷 공유 및 지자체 맞춤형 SaaS 대시보드와 전용 API 연동 지원

3. 중복 다중신고

번화가 위 동일 포트홀에 수백 건의 개별 신고가 발생할 경우 데이터 대조 행정력 낭비 초래

자동 클러스터링

EXIF 좌표 기반 분석으로 반경 내 데이터를 자동 병합(Merge)하여 게시물 중복 방지

4. 비즈니스 모델(BM)

공익 서비스의 특성과 짧은 앱 체류 시간으로 인해 단순 B2C 광고 수익만으로는 자생력 확보가 힘듦

Data B2B 비즈니스

크랙 포인트를 통한 리워드 생태계 구성, 향후 카카오/T맵 등에 Data API 판매 서비스 연계 (B2B)

5. 실질적 해결 권한

민간 기업은 아스팔트 공사를 할 수 없으므로 문제의 완전한 해결을 유도하기 힘듦

양방향 통신 피드백

지자체의 '보수 완료' 상태를 즉각 앱 유저에게 실시간 Push 통보하여 체감 효능감 극대화

해외 벤치마크 — 민간이 먼저 DB를 구축하고, 정부 예산 사업으로 연결되는 구조

미국의 Waze는 시민 제보로 도로 위험 DB를 자체 축적한 뒤 CCP(Connected Citizens Program)로 지자체와 데이터를 교환, 정부 도로 보수 예산 편성의 근거 자료로 활용되고 있습니다. CRACK도 동일하게 민간 AI 데이터를 선행 구축하고, 향후 지자체의 도로 유지보수 예산 사업에 데이터 공급자로 참여하는 B2G 파이프라인을 지향합니다.

Phase 1: 1차 초기 학습 (v5, Bounding Box)

기본 도로 파손 데이터셋(AI-Hub) 100 Epoch 완주 및 하이퍼파라미터 튜닝

Training Metrics

Test Set 없이 Val 10% 할당으로 과적합(Overfitting) 모니터링망 구축

  • 에포크: 총 100 Epoch 완주
  • 검증용(Val) 데이터: 매 에포크당 21,421장 검증
  • 훈련용(Train) 데이터: 매 에포크당 192,798장 학습
  • 100 에포크 누적: 약 74시간(3일) 구동

1차 학습 결과 및 문제점

포트홀 부분에서 높은 성능을 달성했으나, 가장 치명적 위협인 싱크홀(Sinkhole) 데이터가 없는 21만장의 사진이므로, 씽크홀과 포트홀의 학습이 필요했습니다.

실측 정확도 (mAP50) 86.88%
재현율 (Recall) 80.75%
정밀도 (Precision) 82.96%

Phase 2: 2차 싱크홀 통합 보강 학습 (v6)

New Dataset

'Sinkhole' 1,500장 자체 수급

치명적 위험 요소인 씽크홀(지반 침하) 데이터를 웹 스크래핑하여 보강 조치
